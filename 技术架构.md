## 一、整体架构概览

My-Blog 后端采用**领域驱动设计（DDD）**结合**分层架构**构建，严格遵循依赖倒置原则，各层职责清晰，依赖方向单向向内（Interfaces → Application → Domain ← Infrastructure）。


## 三、DDD 分层架构详解

### 3.1 领域层（Domain Layer）

领域层是业务核心，**不依赖任何框架**，只包含纯 Java 对象。

|--------|------|----------|
| User | User.java | 用户账号、认证、资料、密码重置 |
| Article | Article.java | 文章生命周期、计数管理、版本控制 |
| Comment | Comment.java | 评论树、编辑窗口、置顶、点赞 |
| Column | Column.java | 专栏发布、文章管理、学习路径 |
| Topic | Topic.java | 专题管理 |
| Category | Category.java | 分类管理 |
| Tag | Tag.java | 标签管理 |
| Notification | Notification.java | 站内通知、已读状态 |
| Message | Message.java | 私信内容、双向删除 |
| Conversation | Conversation.java | 私信会话、参与者管理 |
| UserFollow | UserFollow.java | 关注关系，复用/重新激活 |
| ArticleLike | ArticleLike.java | 文章点赞，软删除+复用 |
| ArticleFavorite | ArticleFavorite.java | 文章收藏 |
| CommentLike | CommentLike.java | 评论点赞 |
| ColumnSubscription | ColumnSubscription.java | 专栏订阅 |
| Report | Report.java | 举报工作流 |
| AdCampaign | AdCampaign.java | 广告投放、有效期判断 |
| AdminLog | AdminLog.java | 管理员操作审计 |

#### 3.1.2 值对象清单（18 个）

每个聚合根主键对应一个值对象，封装了等值语义（equals/hashCode）：

`UserId` · `ArticleId` · `CommentId` · `ColumnId` · `TopicId` · `TagId` · `CategoryId` · `NotificationId` · `MessageId` · `ConversationId` · `UserFollowId` · `ArticleLikeId` · `ArticleFavoriteId` · `CommentLikeId` · `ColumnSubscriptionId` · `ReportId` · `AdCampaignId` · `Email`（包含格式校验）

#### 3.1.3 枚举值清单

| 枚举 | 可选值 |
|------|--------|
| UserRole | USER, ADMIN |
| UserStatus | NORMAL, DISABLED |
| ArticleStatus | DRAFT, SCHEDULED, PUBLISHED, OFFLINE, DELETED |
| NotificationType | ARTICLE_LIKE, ARTICLE_FAVORITE, ARTICLE_COMMENT, COMMENT_REPLY, COMMENT_LIKE, USER_FOLLOW |

| ReportReasonType | SPAM, INFRINGEMENT, ABUSE, ILLEGAL, OTHER |
| ReportTargetType | ARTICLE, COMMENT |
| ReportResolveAction | REMOVE_CONTENT, DISABLE_USER, DELETE_COMMENT |

#### 3.1.4 工厂方法模式

所有聚合根构造函数均为 `private`，通过两个静态工厂方法实例化：

```java
// 新建场景：由业务规则决定初始值
public static Article create(authorId, title, content, ...);

// 恢复场景：从数据库还原，不触发业务规则
public static Article restore(id, ..., version);
```

这种设计保证了**业务不变式只在 `create()` 时校验**，持久化还原不会触发重复校验。

## 五、安全架构

### 5.1 认证流程

```
客户端请求
     │
     ▼
RequestTimingFilter ──► 注入 traceId 到 MDC
     │
     ▼
JwtAuthenticationInterceptor.preHandle()
     │
     ├── OPTIONS 请求 ──────────────────────────────► 放行（CORS预检）
     │
     ├── 公开接口(isPublicRequest=true) ──► 宽松认证
     │        │                                  │
     │        │                         Token存在 ├─ 解析成功 ──► 注入AuthContext
     │        │                                  └─ 解析失败 ──► 匿名继续
     │        └──────────────────────────────────────────────► 放行
     │
     └── 受保护接口(isPublicRequest=false) ──► 严格认证
              │
              ├─ 无Token ──────────────────────────► 抛 UNAUTHORIZED
              ├─ Token解析失败 ──────────────────────► 抛 UNAUTHORIZED
              ├─ 用户被禁用 ────────────────────────► 抛 FORBIDDEN
              └─ 认证成功 ──────────────────────────► 注入AuthContext 放行
```

### 5.2 JWT 结构

```
Header (Base64URL)        Payload (Base64URL)      Signature (HMAC-SHA256)
┌──────────────────┐      ┌──────────────────────┐  ┌──────────────────────┐
│ {                │      │ {                    │  │ HMAC-SHA256(         │
│   "alg": "HS256",│  .   │   "userId": 1001,    │. │   base64(header) +   │
│   "typ": "JWT"   │      │   "username": "xxx", │  │   "." +              │
│ }                │      │   "role": "USER",    │  │   base64(payload),   │
└──────────────────┘      │   "expireAt": 17...  │  │   secret_key         │
                          │ }                    │  │ )                    │
                          └──────────────────────┘  └──────────────────────┘
```

### 5.3 线程安全保障

```java
// 请求开始：JwtAuthenticationInterceptor.preHandle()
AuthContext.set(jwtPayload);   // ThreadLocal.set()

// 请求处理：Controller / AppService
AuthContext.getCurrentUserId(); // ThreadLocal.get()

// 请求结束：JwtAuthenticationInterceptor.afterCompletion()
AuthContext.clear();            // ThreadLocal.remove() ← 防止线程复用泄露
```

## 六、异步与实时架构

### 6.1 异步任务处理

```
业务操作（如：文章点赞）
     │
     ├── 同步：更新聚合根计数 + 持久化（必须完成）
     │
     └── 异步（taskExecutor 线程池）：
          ├── 发送站内通知
          ├── 更新缓存失效
          └── 记录访问日志
```

### 6.2 SSE 实时推送

```
客户端 EventSource ──► GET /api/notifications/stream?token=xxx
                              │
                              ▼
                       NotificationController
```
