# 核心体验优化计划

## 背景

当前博客功能完整，但三条关键路径存在体验收敛问题：

1. 未登录用户可直接进入写作页，容易先写作后遇到保存/发布拦截。
2. 首页首屏信息偏散，精选区和空模块抢占文章流曝光。
3. 搜索页默认展示全量文章列表，搜索建议和意图引导不够明确。

## 目标

- 未登录用户访问写作页时先完成登录，不展示完整编辑器。
- 首页首屏优先露出分类和文章列表，隐藏无内容模块。
- 搜索页默认展示搜索建议，有搜索意图后再展示结果。
- 仅调整前端体验，不新增后端能力，不改数据库结构。

## 改动范围

### 未登录写作闭环

- 为 `/editor/new` 和 `/editor/:id` 增加登录保护。
- 路由守卫等待 session 初始化完成后再判断登录态。
- 未登录访问编辑器时取消导航，打开登录弹窗。
- 登录成功后跳回原编辑器 URL。
- 编辑器内部保存、发布、定时发布的登录校验继续保留。

### 首页首屏聚焦

- 压缩首页介绍区高度。
- 首页主顺序调整为：紧凑介绍区、分类导航、文章列表、精选推荐。
- 侧栏空数据模块不展示“正在整理中”类占位。
- 侧栏优先展示真实可点内容：推荐专栏、活跃作者、广告。

### 搜索默认态重构

- `/search` 默认不展示全量文章列表。
- 默认态展示最近搜索、热门关键词、热门分类、热门标签。
- 输入关键词、点击建议、使用筛选或切换结果 tab 后进入结果态。
- 结果态保留现有文章、作者、专栏 tab 和移动端筛选抽屉。

## 验收标准

- 未登录访问 `/editor/new` 看不到完整编辑器，并出现登录引导。
- 登录成功后能进入原目标编辑器页面。
- 桌面端首页首屏能看到文章列表开头。
- 移动端首页顶部不拥挤，空专题/空作者模块不出现。
- `/search` 默认进入建议态，不展示 295 篇全量文章列表。
- 搜索关键词后 URL query 同步，结果列表和分页正常。
- 前端 `npm run build` 通过。
