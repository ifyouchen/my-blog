## 2. 实现未登录写作闭环

1. 在 `frontend/src/stores/session.js` 中让 `initializeSession` 幂等：
   - 增加模块级 `initializePromise`。
   - 如果没有 token，直接标记 `state.ready = true`。
   - 如果已有初始化请求，复用同一个 Promise。
   - 初始化成功、失败或清除 session 后都要正确恢复 `state.ready`。
2. 在 `frontend/src/router/index.js` 中为编辑器路由增加：
   - `/editor/new`：`meta.requiresAuth = true`
   - `/editor/:id`：`meta.requiresAuth = true`
3. 在路由守卫中导入 `useSession` 和 `useLoginModal`：
   - 导航开始时继续保留 NProgress。
   - 如果目标路由需要登录，先等待 session 初始化完成。
   - 若未登录，调用 `showLoginModal`，登录成功回调执行 `router.push(to.fullPath)`。
   - 当前导航返回 `{ path: '/', replace: true }`，避免展示编辑器。
4. 保留 `EditorView.vue` 内 `persistArticle` 的 `requireLogin` 校验作为兜底。
5. 在 `EditorView.vue` 初始化发布校验前增加 token 判断：
   - 未登录时不调用 `/articles/validate`。
   - 正常情况下路由守卫会阻止未登录进入编辑器，此判断只作为防御。

## 3. 调整首页首屏结构

1. 在 `frontend/src/views/HomeView.vue` 调整模板顺序：
   - 保留 `HomeIntro`，但作为紧凑介绍。
   - `TopicStrip` 紧跟介绍区。
   - `content-grid` 提前展示 `ArticleFeed` 和 `HomeSidebar`。
   - `FeaturedArticlesStrip` 移到文章列表区域之后，作为次级推荐。
2. `HomeSidebar` 传参保持：
   - `specials="sidebarColumns"`
   - `authors="sidebarAuthors"`
   - `topics` 可以保留但空时不渲染。
3. 在 `frontend/src/components/HomeSidebar.vue` 中：
   - 推荐专栏使用 `props.specials` 渲染，链接到 `/columns/:id`。
   - 活跃作者仅在 `props.authors.length > 0` 时渲染。
   - 热门专题仅在 `props.topics.length > 0` 时渲染。
   - 删除或隐藏“专题内容正在整理中”“作者榜正在生成中”等空占位。
   - 保留 `AdBanner`，其组件自身无广告时不会渲染。
4. 在 `frontend/src/styles/global.css` 中压缩 `.quick-panel`：
   - 减少 padding、gap、阴影视觉重量。
   - 降低标题字号。
   - 统计卡片高度更紧凑。
   - 移动端继续隐藏说明文案，保证首屏能露出内容列表。

## 4. 重构搜索默认态

1. 在 `frontend/src/views/SearchView.vue` 新增计算属性：
   - `hasArticleFilters`：分类、标签、非默认排序、作者关键字、日期、仅看关注任一存在即为 true。
   - `hasSearchIntent`：关键词存在、`hasArticleFilters` 为 true、或 active tab 不是默认文章 tab。
   - `showSearchDefaultState`：`activeTab === 'articles' && !hasSearchIntent`。
2. 修改 `fetchCurrentTab`：
   - 当 `showSearchDefaultState` 为 true 时，不调用 `fetchArticles`。
   - 同时清空 articles、articleTotal，并重置列表请求状态。
   - 有搜索意图时维持现有文章、作者、专栏请求逻辑。
3. 修改模板：
   - 默认态显示搜索建议面板。
   - 建议面板包含最近搜索、热门关键词、热门分类、热门标签。
   - 默认态不显示筛选器、结果数量、广告位、ArticleFeed。
   - 结果态保留现有筛选器、结果数量、ArticleFeed、用户结果、专栏结果。
4. 新增默认态交互方法：
   - 点击关键词：设置 `keyword` 并调用 `runSearch()`。
   - 点击分类：设置 `activeCategory`，清空页码，进入文章结果态。
   - 点击标签：设置 `activeTag`，清空页码，进入文章结果态。
5. 保持移动端筛选抽屉逻辑：
   - 只有结果态且文章 tab 时显示筛选入口。
   - 重置和确认按钮继续使用现有方法。
6. 样式新增：
   - `.search-default-panel`
   - `.search-suggestion-grid`
   - `.search-suggestion-section`
   - `.search-suggestion-chip`
   - 保持蓝白体系，与现有 `keyword-chip`、`tag-row` 风格一致。

## 5. 验证步骤

1. 在 `frontend/` 执行 `npm run build`。
2. 启动前端：`npm run dev -- --host 127.0.0.1`。
3. 验证编辑器：
   - 清空本地 session 后访问 `/editor/new`。
   - 确认看不到编辑器主体，出现登录弹窗。
   - 登录成功后确认跳回 `/editor/new`。
4. 验证首页：
   - 1440px 桌面端首屏能看到文章列表开头。
   - 390px 移动端顶部搜索和写文章不发生明显挤压。
   - 无专题数据时不出现“正在整理中”。
5. 验证搜索：
   - 访问 `/search`，确认展示建议态而不是文章全量列表。
   - 输入 Redis 搜索，确认进入结果态且 URL 包含 keyword。
   - 点击热门分类/标签，确认进入对应结果态。
   - 移动端打开筛选抽屉，确认重置和确认可用。
