一、PRD：内容发现与创作者增长优化（v1.0）
1. 背景与目标
   背景
   当前项目已完成核心阅读/发布/互动流程，并做了可用性优化（空态、移动 TOC、分类折叠等）。但从产品增长角度，仍缺“发现效率 + 创作者经营 + 数据闭环”。.

目标（30天）
首页点击率 CTR 提升 15%

搜索成功率（有结果且有点击）提升 10%

文章二跳率提升 8%

创作者周发文率提升 10%

2. 范围（本期）
   模块A：首页双通道（推荐 / 关注）
   模块B：搜索无结果智能兜底
   模块C：详情页“阅读完成触发推荐”
   模块D：创作者任务中心（Dashboard Action）
3. 用户角色
   游客（未登录）

登录读者

创作者（登录且有发布行为）

管理员（本期不改后台管理逻辑）

4. 详细需求
   A. 首页双通道（推荐 / 关注）
   A1 需求描述
   首页文章流新增 tab：

推荐（默认：游客）

关注（默认：登录用户）

A2 交互规则
游客点击“关注”时触发登录引导

登录用户切换 tab 后，URL query 同步（feedTab=recommend|following）

刷新/分享链接后 tab 状态可恢复

A3 字段定义
feedTab: string, enum[recommend,following], default:

guest: recommend

logged-in: following

A4 状态机
INIT → RESOLVE_DEFAULT_TAB → LOAD_LIST

LOAD_LIST_SUCCESS / LOAD_LIST_EMPTY / LOAD_LIST_ERROR

SWITCH_TAB → LOAD_LIST

A5 埋点事件
home_feed_tab_exposed

props: tab, is_login

home_feed_tab_clicked

props: from_tab, to_tab, is_login

home_feed_list_loaded

props: tab, item_count, latency_ms

A6 验收标准
未登录访问首页默认 推荐

登录访问首页默认 关注

query 变化时列表与 tab 同步

切换 tab 后刷新仍保持一致

B. 搜索无结果智能兜底
B1 需求描述
当用户搜索无结果时，不仅展示空态，还要展示：

相近关键词（最多3个）

热门标签（最多6个）

一键切换到其他 tab（文章/作者/专栏）

B2 字段定义
suggestKeywords: string[]

hotTags: string[]

emptyReason: enum[no_match,filter_too_strict]

B3 状态机
SEARCH_SUBMIT → LOADING

RESULT_EMPTY:

展示 suggestKeywords、hotTags

用户可 applySuggestion or switchTab

B4 埋点
search_empty_exposed

props: tab, keyword, filter_count

search_empty_suggestion_clicked

props: suggested_keyword, origin_keyword

search_empty_switch_tab_clicked

props: from_tab, to_tab

B5 验收标准
无结果时至少展示一个可执行动作

点击建议词后自动发起新搜索

切 tab 后 URL query 同步（你已有 switchTab基础）.

C. 详情页阅读完成触发推荐
C1 需求描述
当阅读进度 >= 70% 时，在正文后显示“继续阅读推荐条”，优先推荐同分类文章。

C2 字段定义
readingProgress: number(0~100)（已存在）

deepReadTriggered: boolean

recommendSource: enum[same_category,related,hot]

C3 状态机
ARTICLE_OPEN → READING

readingProgress >= 70 且 deepReadTriggered=false → SHOW_DEEP_RECOMMEND

C4 埋点
article_deep_read_triggered

props: article_id, progress

article_recommend_click

props: article_id, target_article_id, source

C5 验收标准
达阈值后仅触发一次

推荐条不遮挡正文与评论区

移动端表现正常

D. Dashboard 任务中心
D1 需求描述
在创作台新增“今日建议动作”卡片（最多2条），根据用户状态动态生成：

无文章：去写第一篇

草稿超3天未更新：继续写作

7天未发布：建议发布

D2 字段定义
TaskCard {
taskId: string
type: 'create_first' | 'resume_draft' | 'publish_streak'
title: string
desc: string
ctaText: string
ctaRoute: string
priority: number
}
D3 状态机
DASHBOARD_LOAD → TASK_RULE_EVAL → SHOW_TASKS

D4 埋点
dashboard_task_exposed

dashboard_task_clicked

dashboard_task_completed（可后置）

D5 验收标准
登录创作者必见任务卡（满足条件时）

CTA 可达对应页面

与现有空态 CTA 不冲突。

5. 非功能要求
   兼容移动端（<=980px）

不改后端鉴权模型

不引入阻塞渲染的大体积依赖

6. 风险与依赖
   关注流接口若未准备，先前端 mock fallback

搜索建议词可能依赖后端词库，先做前端轻量规则

e2e 环境需可执行（当前 Playwright 下载受限）

7. 发布计划（建议）
   Sprint 1：A + B

Sprint 2：C + D

Sprint 3：埋点校验与AB复盘
