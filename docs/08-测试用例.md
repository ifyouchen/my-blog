# 博客网站第一版测试用例

## 1. 测试说明

本文档用于验证第一版核心功能是否满足需求。测试重点包括用户认证、文章管理、浏览搜索、评论互动、点赞收藏、权限控制和后台管理。

## 2. 用户认证测试

| 编号 | 测试目标 | 前置条件 | 操作步骤 | 预期结果 |
|---|---|---|---|---|
| TC-AUTH-001 | 用户注册成功 | 用户名和邮箱未被使用 | 打开注册页，输入用户名、邮箱、密码，点击注册 | 注册成功，返回用户基础信息 |
| TC-AUTH-002 | 重复用户名注册失败 | 用户名已存在 | 使用已存在用户名注册 | 注册失败，提示用户名已存在 |
| TC-AUTH-003 | 邮箱格式错误 | 无 | 输入错误邮箱格式注册 | 注册失败，提示邮箱格式错误 |
| TC-AUTH-004 | 用户登录成功 | 用户已注册且状态正常 | 输入正确账号密码登录 | 登录成功，返回 JWT 和用户信息 |
| TC-AUTH-005 | 密码错误登录失败 | 用户已注册 | 输入错误密码登录 | 登录失败，提示账号或密码错误 |
| TC-AUTH-006 | 禁用用户不能登录 | 用户状态为 `DISABLED` | 输入正确账号密码登录 | 登录失败，提示账号已禁用 |
| TC-AUTH-007 | 获取当前用户信息 | 用户已登录 | 请求 `/api/users/me` | 返回当前用户信息 |

## 9. 权限安全测试

| 编号 | 测试目标 | 前置条件 | 操作步骤 | 预期结果 |
|---|---|---|---|---|
| TC-SEC-001 | 无 Token 访问受保护接口 | 未登录 | 请求发文接口 | 返回 `401` |
| TC-SEC-002 | 无效 Token 访问受保护接口 | Token 无效 | 请求当前用户接口 | 返回 `401` |
| TC-SEC-003 | 普通用户访问管理员接口 | 普通用户已登录 | 请求 `/api/admin/users` | 返回 `403` |
| TC-SEC-004 | 用户不能修改他人资料 | 用户 A 已登录 | 修改用户 B 资料 | 返回无权限 |
| TC-SEC-005 | 禁用后旧 Token 不能继续操作 | 用户已登录后被禁用 | 使用旧 Token 发文 | 返回账号禁用或无权限 |

## 10. 验收结论标准

- 所有核心流程用例通过。
- 权限相关用例必须通过。
- 后台管理核心用例必须通过。
- 如存在非阻塞缺陷，需要记录缺陷描述、影响范围和修复计划。
