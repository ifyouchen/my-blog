# 博客网站项目开发规范

> 版本：v0.1.0  
> 说明：本文档是 my-blog 项目的强制开发规范，适用于所有 Spring Boot 后端开发、Vue 3 前端开发、数据库设计、测试和部署运维工作。

## 1. 概述

项目后端采用基于领域驱动设计（DDD）的 Java 单体应用，遵循《阿里巴巴 Java 开发手册》核心规范。前端采用 Vue 3 + Vite，前后端分离。所有代码实现、重构和文档补充必须优先遵循本文档。

## 3. 架构规范

### 3.1 四层架构

```
┌─────────────────────────────────────────────────────────────┐
│ 接口层 interfaces                                           │
│ Controller / DTO / DTO Mapper / 参数校验 / Interceptor       │
├─────────────────────────────────────────────────────────────┤
│ 应用层 application                                          │
│ AppService / Command / Query / Assembler / Event / Listener  │
├─────────────────────────────────────────────────────────────┤
│ 领域层 domain                                               │
│ Aggregate / ValueObject / DomainService / Repository 接口     │
├─────────────────────────────────────────────────────────────┤
│ 基础设施层 infrastructure                                   │
│ Repository 实现 / 持久化 / Mapper / DO / 缓存 / 配置 / 安全   │
└─────────────────────────────────────────────────────────────┘
```

**依赖方向严格单向：**
```
interfaces → application → domain
infrastructure → domain
infrastructure → application
```

### 3.2 包结构

```
com.myblog/
  interfaces/rest/
    controller/           - REST 端点
    dto/                   - 请求/响应 DTO
    mapper/                - DTO <-> Command/Query 转换
  application/
    service/               - 应用服务（用例编排）
    command/               - 命令对象（写）
    query/                 - 查询对象（读）
    assembler/             - DO/领域对象 <-> DTO 转换
    event/                 - 领域事件/应用事件
    listener/              - 事件监听器
    port/                  - 外部能力接口（如邮件）
    cache/                 - 缓存封装
  domain/model/
    aggregate/             - 聚合根
    valueobject/           - 值对象
  domain/service/          - 领域服务
  domain/repository/       - 仓储接口
  infrastructure/repository/persistence/
    entity/                - DO 持久化对象
    mapper/                - MyBatis Mapper 接口
    repository/            - 仓储实现
  infrastructure/security/ - JWT、AuthContext
  infrastructure/config/   - WebMvc、Redisson 等配置
  shared/
    enums/                 - 枚举（UserRole、ArticleStatus 等）
    exception/             - 异常类
    result/                - Result、PageResult

com.myblog.growth/         - 成长模块（同样 DDD 结构，与核心包隔离）
  interfaces/rest/
  application/
  domain/
  infrastructure/
  shared/
```

### 3.3 硬规则

- 领域层不能依赖接口层。
- 领域层不能依赖基础设施实现类。
- Controller 不能直接调用 Mapper。
- Controller 不能包含业务逻辑。
- 应用服务写方法必须使用 `@Transactional(rollbackFor = Exception.class)`。
- 聚合根使用充血模型，状态变更通过业务方法表达（`publish()`、`saveDraft()` 等），禁止暴露 Setter。
- 禁止字段注入，必须使用构造器注入。

## 4. 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 项目名 | 全小写，中划线 | `my-blog` |
| 包名 | 全小写，点号 | `com.myblog.article.domain` |
| 类名 | 大驼峰，名词 | `ArticleService` |
| 方法名 | 小驼峰，动词 | `createArticle()` |
| 变量名 | 小驼峰，名词 | `articleId` |
| 常量 | 全大写，下划线 | `DEFAULT_PAGE_SIZE` |
| 枚举 | 大驼峰，枚举值全大写 | `ArticleStatus { PUBLISHED }` |
| 数据库表 | 全小写，下划线，`blog_` 前缀 | `blog_article` |
| 数据库字段 | 全小写，下划线 | `created_at` |

## 5. Java 编码规范

- 4 空格缩进，禁止 Tab。
- 单行不超过 120 字符。
- 左大括号不换行，右大括号换行。
- 禁止通配符导入（`java.util.*`）。
- 所有 `public` 类和 `public` 方法必须有 Javadoc（包含 `@author` 和 `@since`）。
- 强制构造器注入，禁止 `@Autowired` 字段注入。
- 包装类型比较必须使用 `.equals()`，禁止 `==`。
- 字符串常量比较时，常量放左边：`"PUBLISHED".equals(status)`。
- 集合必须指定泛型，返回空集合而非 `null`。
- `if`/`while`/`for` 必须使用大括号。
- 嵌套层级不超过 3 层。
- 线程池必须显式创建 `ThreadPoolExecutor`，禁止 `Executors` 快捷方法。

## 6. DDD 编码规范

### 6.1 聚合根

- 聚合根负责维护聚合内部不变性。
- 充血模型，业务行为写在领域对象中。
- 禁止随意修改状态的 Setter。
- 外部引用聚合优先使用 ID，不直接持有其他聚合根对象。
- 状态变更必须通过业务方法表达：`publish()`、`offline()`、`delete()`。
- 领域事件由聚合根注册，应用服务负责发布。

### 6.2 值对象

- 必须不可变，字段优先使用 `final`。
- 通过构造器或静态工厂方法完成校验。
- 金额使用 `BigDecimal`，禁止 `double`/`float`。

### 6.3 领域服务

- 处理无法自然归属于单个实体或值对象的领域逻辑。
- 只依赖其他领域对象、领域服务或仓储接口。
- 禁止依赖 Controller、DTO、Mapper、DO。

### 6.4 应用服务

- 负责用例编排、事务边界、权限上下文协调和领域事件发布。
- 不写核心业务规则。
- 变更方法必须加 `@Transactional(rollbackFor = Exception.class)`。
- 查询类方法优先走 QueryService。

### 6.5 仓储

- 接口定义在领域层，实现在基础设施层。
- 返回领域对象，不返回 DO。
- 可能为空的查询返回 `Optional`。
- 实现负责 DO 与领域对象的转换。

## 7. Controller 规范

- Controller 只处理 HTTP 入参、参数校验、DTO 转换和响应包装。
- 禁止直接访问 Mapper。
- 禁止直接处理领域规则。
- 请求 DTO 使用 JSR-380 注解校验（`@NotBlank`、`@Size`、`@Email` 等）。
- 返回统一使用 `Result<T>` 或 `PageResult<T>`。

## 8. 数据库规范

### 8.1 表设计

- 所有表使用 `utf8mb4` 字符集、InnoDB 引擎。
- 所有业务表使用 `blog_` 前缀（成长模块使用 `user_` 和 `growth_` 系列）。
- 所有表必须包含：

```sql
id          BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键ID',
created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
updated_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
deleted_at  DATETIME DEFAULT NULL COMMENT '删除时间(软删除)',
version     INT NOT NULL DEFAULT 0 COMMENT '版本号(乐观锁)'
```

### 8.2 软删除

- 禁止物理删除，全部使用 `deleted_at IS NULL` 条件过滤。
- 所有查询必须过滤 `deleted_at IS NULL`。

### 8.3 MyBatis 规范

- Mapper XML 必须定义 `BaseResultMap` 和 `Base_Column_List`。
- 查询必须过滤 `deleted_at IS NULL`。
- 更新必须处理乐观锁 `version`。
- SQL 必须参数化，禁止字符串拼接。
- Mapper 只做数据访问，不包含业务逻辑。

### 8.4 索引规范

- 主键必须存在。
- 唯一约束建立唯一索引。
- 索引名格式：`idx_表名_字段名`，唯一索引：`uk_表名_字段名`。
- 联合索引中区分度高的字段放左边。
- 全文索引使用 ngram 解析器（支持中文）。

## 9. 异常处理

- 禁止直接抛出裸 `RuntimeException`。
- 业务异常继承 `BusinessException`，领域异常继承 `DomainException`。
- 全局异常处理器统一返回 `Result<T>`。
- 业务异常记录 WARN 日志，系统异常记录 ERROR 日志（含堆栈）。
- 禁止 `e.printStackTrace()`。

## 10. 日志规范

- 门面使用 SLF4J，实现使用 Logback。
- 必须使用 `{}` 占位符，禁止字符串拼接。
- 敏感信息必须脱敏。
- 日志文件至少保存 30 天。

```yaml
logging:
  level:
    root: INFO
    com.myblog: DEBUG
    com.myblog.application.cache: DEBUG
  file:
    name: logs/application.log
    max-size: 100MB
    max-history: 30
```

## 11. 安全规范

- 所有外部输入必须校验（JSR-380）。
- 密码使用 BCrypt 哈希存储。
- JWT 密钥通过环境变量注入，不提交仓库。
- 日志禁止打印密码、Token、手机号、邮箱等敏感信息。
- SQL 必须参数化。
- 文件上传校验类型、大小（20MB/文件，22MB/请求）和频率限制。
- 接口限流：默认每 IP 每分钟 20 次，图片上传 10 次/60s，文件上传 5 次/60s。
- 使用 AuthContext（ThreadLocal）传递用户身份，请求结束后必须 clear。
- 用户状态每次请求校验（Redis 缓存 5 分钟），禁用用户即使 Token 有效也不能操作。
- 前端权限控制仅用于体验，后端做最终校验。

## 12. 性能规范

- 事务粒度尽量小，事务内禁止远程调用。
- 循环内禁止数据库查询（避免 N+1）。
- 批量写入使用批处理，单批次不超过 1000 条。
- 深分页使用游标或延迟关联优化。
- 热点数据使用两级缓存（Caffeine L1 + Redis L2），通过 Redisson RTopic 广播失效。
- 异步处理必须使用自定义线程池（邮件队列等）。
- 连接池使用 HikariCP，配置合理连接数。
- SSE 端点（`/api/notifications/stream`、`/api/messages/stream`）使用独立线程池。

## 13. 测试规范

- 单元测试使用 JUnit 5 + AssertJ。
- 测试方法名描述业务场景，遵循 Given / When / Then 结构。
- 领域模型核心业务方法必须有单元测试。
- 集成测试优先使用真实 MySQL 或 Testcontainers。
- E2E 测试使用 Playwright，覆盖核心用户流程。
- 运行：`cd backend && mvn test`（后端），`cd frontend && npm run test:e2e`（前端 E2E）。

## 14. 成长模块规范

- 所有成长模块代码在 `com.myblog.growth.*` 包下。
- 博客核心包 `com.myblog.*` 禁止 import `com.myblog.growth.*`。
- 成长模块可通过事件监听（`@EventListener`）响应博客侧行为。
- 成长模块可注入 UserRepository 读取用户基本信息。
- 成长模块不能直接调用博客侧的 AppService。
- 成长模块不能直接操作博客侧数据库表（读 `blog_article` 特定字段除外）。

## 15. 前端规范

- 使用 Vue 3 Composition API + `<script setup>`。
- 路由定义在 `src/router/index.js`，使用 `meta.requiresAuth` 和 `meta.requiresAdmin` 控制访问。
- API 模块在 `src/api/` 下，使用原生 `fetch()`（非 Axios），返回 unwrapped data。
- 状态管理使用 `reactive()` 组合式 store（非 Pinia）。
- 页面级组件在 `src/views/`，通用组件在 `src/components/`。
- 编辑器使用 TipTap + 代码高亮（highlight.js/lowlight）+ DOMPurify XSS 过滤。
- 构建：`npm run build`。
