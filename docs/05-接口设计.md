# 博客网站接口设计

> 版本：v0.1.0  
> 说明：本文档基于实际代码生成，列出全部 REST API 端点（约 330+）。接口按 Controller 分组，覆盖核心模块和成长模块。

## 1. 接口规范

- **基础路径**：`/api`
- **认证方式**：自定义 JWT（HMAC-SHA256），请求头 `Authorization: Bearer <token>` 或查询参数 `?token=<token>`
- **内容类型**：`application/json`（文件上传使用 `multipart/form-data`）

### 1.1 统一返回结构

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

### 1.2 分页返回结构

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "items": [],
    "page": 1,
    "pageSize": 10,
    "total": 0
  }
}
```

### 1.3 错误码

| 错误码 | 说明 |
|------:|------|
| 0 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或 Token 无效 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 409 | 数据冲突 |
| 429 | 请求过于频繁（限流） |
| 500 | 服务器内部错误 |

## 16. 文件上传

### 16.1 上传图片

`POST /api/uploads/images`

- 权限：登录（限流：每 IP 每 60 秒 10 次）
- 请求类型：`multipart/form-data`，字段名 `file`
- 返回：`{"url": "/uploads/2026/04/image.png"}`

### 16.2 上传文件

`POST /api/uploads/files`

- 权限：登录（限流：每 IP 每 60 秒 5 次）
- 请求类型：`multipart/form-data`，字段名 `file`

### 16.3 获取预签名 URL

`POST /api/uploads/presigned-url`

- 权限：登录（限流）
- 请求体：`{"fileName": "image.png", "scope": "cover"}`（scope: cover/avatar/chat）
- 返回：`{"presignedUrl": "...", "url": "...", "thumbnailUrl": "..."}`
- 说明：返回 COS 预签名直传 URL，前端直接 PUT 上传，不经过后端。如 COS 未配置则降级为代理上传。
