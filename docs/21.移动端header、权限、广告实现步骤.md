1. 移动端 Header

目标文件：

SiteHeader.vue、global.css


在 SiteHeader.vue 新增状态：


mobileSearchOpen = ref(false)

继续复用现有 keyword




新增方法：


openMobileSearch()：打开移动搜索面板，关闭用户菜单/通知菜单

closeMobileSearch()：关闭面板

submitSearch() 改成搜索后同时关闭 mobileSearchOpen 和 mobileMenuOpen

建议搜索时 keyword.value.trim()，空关键词跳 /search，非空跳 /search?keyword=...




在 header actions 里增加移动端搜索按钮：


class：mobile-search-toggle

aria-label="搜索"

图标用简单放大镜 SVG 或现有图标风格

点击调用 openMobileSearch




小屏隐藏顶部写文章按钮：


给顶部写文章按钮加可识别类，比如 desktop-write-action

在 @media (max-width: 760px) 下隐藏 .desktop-write-action

抽屉里的“写文章”保留




修复抽屉写文章点击 bug：


当前 @click="writeArticle; mobileMenuOpen = false" 没有真正调用 writeArticle

改成 @click="mobileMenuOpen = false; writeArticle()"




添加移动端搜索面板模板：


建议用 Teleport to="body"

v-if="mobileSearchOpen"

包含遮罩、面板、输入框、取消按钮、搜索按钮

form 提交走 submitSearch

输入框加 data-testid="mobile-site-search-input"

搜索按钮加 data-testid="mobile-site-search-submit"




CSS 调整：


默认隐藏 .mobile-search-toggle 和 .mobile-search-panel

max-width: 760px 下显示 .mobile-search-toggle

max-width: 760px 下隐藏 .search-box

header 改为 grid-template-columns: auto minmax(0, 1fr) auto

.header-actions 只保留搜索按钮、菜单按钮

移动搜索面板固定顶部，宽度 100%，高度不必全屏，建议顶部浮层即可
