# 经验与积分模块 — 架构规范

> **核心约束：经验 / 积分 / 解锁是独立的业务模块，在同一个 Spring Boot 工程（`my-blog/backend`）内以独立包划分，不得与现有博客代码耦合。**

---

## 1. 模块定位

| 模块 | Java 根包 | 说明 |
|------|-----------|------|
| 博客核心 | `com.myblog.*` | 现有博客功能，**不做任何侵入式修改** |
| 成长模块 | `com.myblog.growth.*` | 经验、等级、积分、签到、邀请、解锁、分账 |

同一工程、同一数据库（`my_blog`），但**包隔离、不互相 import**。

---

---

## 3. 分层依赖规则（与博客核心一致）

```
interfaces → application → domain
infrastructure → domain
infrastructure → application
```

**禁止**：
- `domain` 层 import `com.myblog.*`（博客核心类）
- `application` 层直接调用博客的 Mapper 或 Repository
- Controller 层包含任何业务逻辑

---

## 4. 跨模块通信协议

### 4.1 博客核心 → 成长模块（行为事件触发）

博客侧代码在发生行为时，通过 **Spring `ApplicationEventPublisher`** 发布事件。
成长模块通过 `@EventListener` 订阅，**不在博客代码里 import 成长模块的类**。

```
博客侧发布（现有代码，已有的事件）：
  ArticleLikedEvent      → 点赞
  ArticleViewedEvent     → 阅读
  CommentCreatedEvent    → 评论
  ArticleFavoritedEvent  → 收藏
  UserFollowedEvent      → 关注

成长模块订阅（com.myblog.growth.application.listener.*）：
  @EventListener(ArticleLikedEvent.class)
  void onArticleLiked(ArticleLikedEvent event) { ... }
```

博客侧**只发事件，不感知成长模块**；成长模块**只监听，不修改博客代码**。

### 4.2 成长模块查询用户信息

成长模块需要校验 `userId` 合法性时，通过 **注入博客域的 `UserRepository` 接口**（接口在 `com.myblog.domain.repository` 包，实现在 infrastructure 层，不存在循环依赖）。

```java
// 允许：依赖博客域的 Repository 接口（纯接口，无业务逻辑）
@Autowired
private com.myblog.domain.repository.UserRepository userRepository;
```

这是唯一允许的跨模块依赖，且只限于**查用户基本信息**。

### 4.3 前端调用

前端统一通过 `/api/growth/...`、`/api/points/...`、`/api/articles/{id}/unlock` 调用成长模块接口。
Vite 代理配置不变（仍指向同一个 8080 端口）。

## 5. 数据库约定

- 成长模块的表与博客核心的表在**同一数据库** `my_blog`
- 建表 SQL 追加到 `backend/src/main/resources/db/schema.sql`，或单独放 `db/growth/` 子目录
- 表名前缀建议：
  - 经验/等级：`user_growth_*`、`growth_rule_*`、`level_*`、`event_consume_*`、`daily_reward_*`
  - 积分：`user_point_*`、`point_rule_*`、`admin_point_*`、`sign_in_*`、`invite_*`
  - 解锁/分账：`article_unlock_*`、`revenue_share_*`
- 全部表包含：`deleted_at`（软删除）、`version`（乐观锁）、`created_at`、`updated_at`

## 6. 代码示例：模块边界

```java
// ✅ 正确：成长模块监听博客事件，博客侧代码无变化
package com.myblog.growth.application.listener;

@Component
public class GrantExpOnBehaviorEventHandler {

    @EventListener
    public void onArticleLiked(ArticleLikedEvent event) {   // 博客侧事件
        expGrantAppService.grantExpForBehavior(...);
    }
}

// ❌ 错误：在博客的 ArticleLikeAppService 里直接调用成长模块
// articleLikeAppService → expGrantAppService  （禁止！）
```
