## 3. 业务规则实现说明

### 3.1 分账比例计算

```
默认比例：平台 50% : 作者 50%

platformPoints = totalPoints × platformRatio（向下取整）
authorPoints   = totalPoints - platformPoints  （作者拿余量，保证守恒）

示例：totalPoints=51，比例 50:50
  platformPoints = floor(51 × 0.5) = 25
  authorPoints   = 51 - 25 = 26
```

**守恒验证**：`platformPoints + authorPoints == totalPoints`（任何订单均满足）

### 3.2 分账比例配置

- 在 `point_rule_config` 中新增 `source_type=UNLOCK_SHARE_RATIO`，`point_amount` 字段存平台分成比例（百分比整数，如 50 = 50%）
- 默认 50:50，管理端可修改

### 3.3 创建分账记录时机

由 P3-T1 的 `ArticleUnlockAppService` 在解锁事务中调用 `RevenueShareAppService.createShareRecord()`，在**同一事务**内完成。`RevenueShareAppService` 不自己开事务（`REQUIRED` 传播级别）。

### 3.4 对账闭环

任意一笔解锁订单满足：
```
article_unlock_order.points_cost
  = revenue_share_journal.total_points
  = revenue_share_journal.platform_points + revenue_share_journal.author_points
  = user_point_journal WHERE biz_no = orderNo AND delta < 0 AND source_type = 'UNLOCK' 的 |delta|
```

## 5. 风险与限制

- 分账比例修改不会影响历史记录（`revenue_share_journal.share_ratio` 已快照当时比例）
- 作者积分不实际转账到作者账户（当前阶段），`revenue_share_journal` 仅作对账记录；如需真实入账，后续版本扩展 `PointAppService.addPoints(authorId, authorPoints, "REVENUE_SHARE", orderNo)`
