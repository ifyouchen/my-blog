## 5. 业务规则实现说明

### 幂等键
- 每笔流水的 `biz_no` 唯一索引保证幂等：重复入账时 `INSERT IGNORE` 或捕获唯一键异常直接返回成功

### 禁负策略
- 扣减前检查：`balance - |delta| >= 0`，否则抛 `BusinessException(POINT_INSUFFICIENT)`
- 数据库层 `CHECK (balance >= 0)` 兜底（MySQL 8.0.16+）

### 事务边界
- `addPoints()` / `deductPoints()`：`@Transactional(rollbackFor = Exception.class)`
- 流水 INSERT + 账户 UPDATE 在同一事务内

### 并发安全
- 账户更新使用乐观锁：`UPDATE user_point_account SET balance=balance+?, version=version+1 WHERE user_id=? AND version=?`
- 失败时重试最多 3 次，超过抛 `ConcurrentModificationException`

## 6. 测试验收点

| 场景 | 预期结果 |
|------|---------|
| 100 并发同时扣减 | balance 精确，无超扣，无负数 |

## 8. 风险与限制

- `user_point_journal` 不可物理删除，历史流水为审计依据
- 管理员调分必须填写 `reason` 且记录 `operator_user_id`，便于审计回溯
- 积分余额不跨用户转移，禁止实现转账功能
