```sql
CREATE TABLE IF NOT EXISTS `sign_in_record` (
    `id`               bigint unsigned NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    `user_id`          bigint unsigned NOT NULL COMMENT '用户ID',
    `sign_date`        date            NOT NULL COMMENT '签到日期',
    `consecutive_days` int unsigned    NOT NULL DEFAULT 1 COMMENT '连续签到天数',
    `points_granted`   int             NOT NULL DEFAULT 0 COMMENT '本次发放积分',
    `created_at`       datetime        NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `deleted_at`       datetime        NULL DEFAULT NULL,
    `version`          int             NOT NULL DEFAULT 0,
    PRIMARY KEY (`id`) USING BTREE,
    UNIQUE INDEX `uk_user_sign_date` (`user_id`, `sign_date`) USING BTREE,
    INDEX `idx_user_date_desc` (`user_id`, `sign_date` DESC) USING BTREE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户签到记录';
```

## 3. 业务规则实现说明

### 3.1 连续签到阶梯奖励

| 连续天数 | 基础积分 | 附加奖励 | 总计 |
|---------|---------|---------|------|
| 1~2 天  | 10 | 0  | 10 |
| 3~4 天  | 10 | 5  | 15 |
| 5~6 天  | 10 | 8  | 18 |
| 7~13 天 | 10 | 10 | 20 |
| 14~29 天| 10 | 20 | 30 |
| 30+ 天  | 10 | 50 | 60 |

### 3.2 断签重算规则

```
查最近一条 sign_in_record（deleted_at IS NULL，sign_date 最大）：
  若 sign_date = today - 1 day  → consecutive_days = last.consecutive_days + 1
  若 sign_date = today          → 已签到，返回 400
  其他（断签）                  → consecutive_days = 1
```

### 3.3 幂等键

- 数据库唯一索引 `uk_user_sign_date(user_id, sign_date)` 是幂等核心
- 业务流程：先 SELECT 当日记录，存在则返回 400；INSERT 失败（唯一键冲突）也返回 400

### 3.4 事务边界

```
SignInAppService.signIn()  @Transactional(rollbackFor = Exception.class)
  ├── 查今日是否已签到 → 存在则返回 400
  ├── 查最近签到记录 → 计算 consecutive_days
  ├── SignInDomainService.calcReward(consecutiveDays) → pointsGranted
  ├── INSERT sign_in_record
  └── PointAppService.addPoints(userId, pointsGranted, "SIGN_IN",
                                 "signin-{userId}-{signDate}")
```

`PointAppService.addPoints` 内部也在事务中执行，此处通过同一事务传播（`REQUIRED`）保证原子性。

## 4. 测试验收点

| 场景 | 预期结果 |
|------|---------|
| 同日重复签到 | 返回 400，积分不重复入账 |
| 连续 3 天签到 | 第 3 天 pointsGranted = 15 |
| 断签后重新签到 | consecutive_days 重置为 1，pointsGranted = 10 |
| 连续 7 天签到 | 第 7 天 pointsGranted = 20，rewardTier = WEEK |
| 日历查询 2026-05 | 返回本月已签到日期列表，todaySigned 正确 |

## 5. 风险与限制

- 时区问题：`sign_date` 以服务器本地日期（Asia/Shanghai）为准，需确保数据库和 JVM 时区一致
- 奖励阶梯若需调整，建议抽入 `point_rule_config` 表，当前版本写在 `SignInDomainService` 常量中，下版本可配置化

---

## 6. 回滚方案

- **配置回滚**：向 `point_rule_config` 中 `source_type=SIGN_IN` 记录设置 `enabled=0`，签到接口调用 `addPoints` 时检查规则，自动跳过发分
- **代码回滚**：注释掉 `SignInController` 的 `@RestController` 注解，接口下线
- **SQL 回滚**：`DROP TABLE IF EXISTS sign_in_record`（已签到数据丢失，仅在回退版本时使用）
