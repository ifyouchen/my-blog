## 3. 接口调用流程

### 3.1 详情页加载

```
进入文章详情页
    │
    ├── 调用 GET /api/articles/{id}           → 获取文章基本信息（含 needUnlock/unlockPointPrice）
    │
    ├── 若 needUnlock=true 且已登录
    │       └── 调用 GET /api/articles/{id}/unlock-status  → 获取 unlocked/currentBalance
    │
    ├── 若 unlocked=true   → 渲染完整正文
    └── 若 unlocked=false  → 渲染遮罩 + 解锁引导区
```

### 3.2 解锁流程

```
用户点击「立即解锁」
    │
    ├── growthStore.balance >= unlockPointPrice
    │       └── 弹出确认弹窗（余额充足）
    │           └── 用户点击「确认解锁」
    │                   └── POST /api/articles/{id}/unlock
    │                           ├── 成功 → 关闭弹窗 → 刷新文章详情（完整正文）
    │                           │         → growthStore.refreshBalance()
    │                           └── 失败（积分不足）→ 弹出提示
    │
    └── growthStore.balance < unlockPointPrice
            └── 直接弹出「积分不足」提示
```

## 4. API 封装（`frontend/src/api/growth.js`）

```javascript
import request from './request'

// 查询我的积分账户
export const getMyPointAccount = () =>
  request.get('/api/points/account')

// 查询文章解锁状态
export const getArticleUnlockStatus = (articleId) =>
  request.get(`/api/articles/${articleId}/unlock-status`)

// 解锁文章
export const unlockArticle = (articleId) =>
  request.post(`/api/articles/${articleId}/unlock`)

// 查询成长信息（等级/经验）
export const getMyGrowthInfo = () =>
  request.get('/api/growth/me')

// 签到
export const signIn = () =>
  request.post('/api/points/sign-in')

// 查询签到日历
export const getSignInCalendar = (month) =>
  request.get('/api/points/sign-in/calendar', { params: { month } })
```

## 5. Pinia Store（`frontend/src/stores/growth.js`）

```javascript
import { defineStore } from 'pinia'
import { getMyPointAccount } from '../api/growth'

export const useGrowthStore = defineStore('growth', {
  state: () => ({
    balance: 0,
    totalEarned: 0,
    currentLevel: 1,
    levelName: '',
    loaded: false
  }),
  actions: {
    async refreshBalance() {
      const res = await getMyPointAccount()
      if (res.code === 0) {
        this.balance = res.data.balance
        this.totalEarned = res.data.totalEarned
        this.loaded = true
      }
    }
  }
})
```

## 7. 风险与限制

- 未解锁时后端**不返回正文内容**（接口层过滤），前端遮罩仅作 UI 引导，无安全意义
- `unlockPointPrice` 展示应与后端保持实时同步，建议每次进入详情页重新请求 unlock-status
- 解锁状态应在用户登录后首次加载，避免频繁请求

---

## 8. 回滚方案

- **前端回滚**：在 `ArticleDetail.vue` 中移除锁态逻辑分支，将 `needUnlock` 判断短路为 `false`
- 不涉及数据库变更，前端代码回滚不影响已有解锁数据
