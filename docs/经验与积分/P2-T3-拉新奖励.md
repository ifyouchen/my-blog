```sql
-- 邀请关系表
CREATE TABLE IF NOT EXISTS `invite_relation` (
    `id`              bigint unsigned NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    `inviter_user_id` bigint unsigned NOT NULL COMMENT '邀请人用户ID',
    `invitee_user_id` bigint unsigned NOT NULL COMMENT '被邀请人用户ID',
    `reward_status`   varchar(20)     NOT NULL DEFAULT 'PENDING' COMMENT '奖励状态：PENDING/GRANTED/SKIPPED',
    `reward_granted_at` datetime      NULL DEFAULT NULL COMMENT '奖励发放时间',
    `trigger_biz_no`  varchar(128)    NULL DEFAULT NULL COMMENT '触发奖励的业务单号',
    `skip_reason`     varchar(500)    NULL DEFAULT NULL COMMENT '跳过原因（风控/已奖励等）',
    `created_at`      datetime        NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `updated_at`      datetime        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    `deleted_at`      datetime        NULL DEFAULT NULL,
    `version`         int             NOT NULL DEFAULT 0,
    PRIMARY KEY (`id`) USING BTREE,
    UNIQUE INDEX `uk_invitee` (`invitee_user_id`) USING BTREE,
    INDEX `idx_inviter_status` (`inviter_user_id`, `reward_status`, `deleted_at`) USING BTREE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='邀请关系表';
```

> `uk_invitee(invitee_user_id)` — 一个被邀请人只能有一个邀请人，同时防止重复奖励。

## 2. 接口清单

### 2.1 触发拉新奖励

**POST** `/api/open/invite/reward/trigger`
**Auth**：无需登录（开放接口，需验签）

> 该接口由注册流程完成后内部调用，或由博客侧通过 Spring 事件驱动。

#### 请求示例

```json
{
  "inviterUserId": 1001,
  "inviteeUserId": 2005,
  "inviteCode": "INV-ABC123",
  "triggerType": "REGISTER_COMPLETE",
  "clientIp": "192.168.1.100",
  "deviceId": "device-fingerprint-xyz"
}
```

#### 响应示例 — 奖励成功

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "inviterUserId": 1001,
    "inviteeUserId": 2005,
    "pointsGranted": 50,
    "rewardStatus": "GRANTED"
  }
}
```

#### 响应示例 — 已奖励（幂等）

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "rewardStatus": "GRANTED",
    "message": "已奖励，不重复发放"
  }
}
```

#### 响应示例 — 风控拦截

```json
{
  "code": 400,
  "message": "风控拦截：同 IP 注册超出频次限制",
  "data": null
}
```

#### 错误码

| code | 说明 |
|------|------|
| 400 | 风控拦截 / 邀请关系不合法 |
| 404 | inviterUserId 或 inviteeUserId 不存在 |

## 3. 业务规则实现说明

### 3.1 发奖条件

```
被邀请人必须满足以下条件才发奖：
  1. 完成注册（triggerType = REGISTER_COMPLETE）
  2. 未被风控拦截
  3. invite_relation 不存在该 invitee（uk_invitee 唯一约束防止重复）
```

### 3.2 风控开关（InviteRiskControlService）

| 风控维度 | 规则 | 可配置 |
|---------|------|------|
| 同 IP 频次 | 同一 IP 24h 内注册 ≤ 3 人 | ✅ |
| 设备指纹 | 同 deviceId 只允许 1 次注册 | ✅ |
| 邀请人上限 | 单人每日最多邀请奖励 10 次 | ✅ |
| 风控总开关 | `point_rule_config.source_type=INVITE` 的 `enabled` 字段 | ✅ |

### 3.3 幂等键

- `uk_invitee(invitee_user_id)`：数据库唯一约束，同一 invitee 只入一条记录
- `trigger_biz_no`：对应 `user_point_journal.biz_no`，格式为 `invite-{inviterUserId}-{inviteeUserId}`

### 3.4 事务边界

```
InviteRewardAppService.triggerReward()  @Transactional(rollbackFor = Exception.class)
  ├── 校验 inviter/invitee 用户存在（调 UserRepository，只读）
  ├── InviteRiskControlService.check(ip, deviceId, inviterUserId) → 风控
  ├── SELECT invite_relation WHERE invitee_user_id = ? FOR UPDATE
  │       └── 若已存在 → 返回幂等成功
  ├── INSERT invite_relation(PENDING)
  ├── PointAppService.addPoints(inviterUserId, 50, "INVITE", bizNo)
  └── UPDATE invite_relation SET reward_status = GRANTED
```

## 4. 测试验收点

| 场景 | 预期结果 |
|------|---------|
| 重复触发同一 invitee | 第 2 次返回 GRANTED（幂等），积分不重复入账 |
| 同 IP 超出频次 | 返回风控拦截错误 |
| 风控开关关闭 | 所有触发请求返回 400 |
| 邀请成功后查 summary | totalInvited +1，rewardGrantedCount +1 |
| 异常场景查审计 | `invite_relation` 表留有完整记录，含 skip_reason |

---

## 5. 风险与限制

- 风控规则数据存 Redis，未引入 Redis 则退化为数据库频次查询（性能低）
- `invitee_user_id` 唯一约束意味着用户只能被邀请一次，若需支持多次邀请场景需调整设计

---

## 6. 回滚方案

- **配置回滚**：`UPDATE point_rule_config SET enabled=0 WHERE source_type='INVITE'`，触发接口将直接跳过发奖
- **代码回滚**：注释 `InviteController` 的 `@RestController`
- **SQL 回滚**：`DROP TABLE IF EXISTS invite_relation`
