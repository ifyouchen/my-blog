# 博客网站第一版部署说明

## 1. 部署目标

第一版采用前后端分离部署：

- 前端 Vue 3 构建为静态资源。
- 后端 Spring Boot 打包为可运行服务。
- 数据库使用 MySQL。
- 生产环境建议使用 Nginx 托管前端资源并反向代理后端接口。

## 2. 环境要求

| 依赖 | 建议版本 | 说明 |
|---|---|---|
| JDK | 8 | 运行 Spring Boot 2.7.18 |
| Maven | 3.8+ | 后端依赖管理和打包 |
| Node.js | 18+ | 前端开发和构建 |
| npm | 9+ | 前端包管理 |
| MySQL | 8.0+ | 数据库 |
| Redis | 6.0+ | 缓存和后续会话扩展 |
| Nginx | 1.20+ | 生产环境前端托管和反向代理 |

## 3. 环境变量

后端建议使用环境变量或配置文件管理敏感信息：

```properties
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_DATABASE=my_blog
MYSQL_USERNAME=root
MYSQL_PASSWORD=123456
REDIS_HOST=192.168.80.128
REDIS_PORT=6379
REDIS_PASSWORD=123456
JWT_SECRET=replace_with_a_long_random_secret
JWT_EXPIRE_SECONDS=86400
UPLOAD_DIR=/data/my-blog/uploads
```

注意：

- `JWT_SECRET` 生产环境必须使用足够长的随机字符串。
- `UPLOAD_DIR` 需要保证后端服务进程有读写权限。
- 数据库密码不要提交到公开仓库。

## 4. 数据库初始化

当前后端已配置 MySQL 自动建库和自动建表：

- 数据库名：`my_blog`
- 建表脚本：`backend/src/main/resources/db/schema.sql`
- MySQL 连接参数：`createDatabaseIfNotExist=true`

如果生产环境不允许应用账号创建数据库，可手动创建数据库：

```sql
CREATE DATABASE my_blog DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

创建数据库用户，可选：

```sql
CREATE USER 'my_blog'@'%' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON my_blog.* TO 'my_blog'@'%';
FLUSH PRIVILEGES;
```

## 5. 后端本地启动

进入后端目录：

```bash
cd backend
```

安装依赖并启动：

```bash
mvn spring-boot:run
```

后端默认地址建议：

```text
http://localhost:8080
```

健康检查接口可后续设计为：

```text
GET /api/health
```

## 6. 前端本地启动

进入前端目录：

```bash
cd frontend
```

安装依赖：

```bash
npm install
```

启动开发服务：

```bash
npm run dev
```

前端默认地址建议：

```text
http://localhost:5173
```

开发环境中，前端可通过代理将 `/api` 请求转发到 `http://localhost:8080`。

## 7. 前端打包

```bash
cd frontend
npm run build
```

构建产物通常位于：

```text
frontend/dist/
```

将 `dist` 目录部署到 Nginx 静态资源目录。

## 8. 后端打包

```bash
cd backend
mvn clean package -DskipTests
```

构建产物通常位于：

```text
backend/target/*.jar
```

运行后端服务：

```bash
java -jar target/my-blog.jar
```

生产环境建议通过 systemd、Docker 或进程管理工具托管后端服务。

## 9. Nginx 配置示例

```nginx
server {
    listen 80;
    server_name example.com;

    root /var/www/my-blog/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header Authorization $http_authorization;
    }

    location /uploads/ {
        alias /data/my-blog/uploads/;
    }
}
```

## 10. 生产部署步骤

1. 安装 JDK、Node.js、MySQL、Nginx。
2. 创建 MySQL 数据库和用户。
3. 执行数据库初始化脚本。
4. 修改后端生产配置。
5. 打包后端并启动 Spring Boot 服务。
6. 打包前端并复制到 Nginx 静态资源目录。
7. 配置 Nginx 反向代理 `/api`。
8. 配置上传目录访问。
9. 重启 Nginx。
10. 验证首页、登录、文章列表和后台接口。

## 11. 常见问题

### 11.1 端口被占用

现象：前端或后端启动失败，提示端口已被占用。

处理：

- 前端修改 Vite 开发端口。
- 后端修改 `server.port`。
- 或停止占用端口的进程。

### 11.2 数据库连接失败

现象：后端启动失败，提示无法连接 MySQL。

处理：

- 检查 MySQL 是否启动。
- 检查数据库地址、端口、用户名和密码。
- 检查数据库是否已创建。
- 检查服务器防火墙和 MySQL 访问权限。

### 11.3 跨域请求失败

现象：浏览器控制台出现 CORS 错误。

处理：

- 开发环境配置后端 CORS。
- 允许前端地址访问。
- 允许 `Authorization` 请求头。
- 生产环境优先通过 Nginx 同域代理解决。

### 11.4 JWT 过期

现象：请求受保护接口返回 `401`。

处理：

- 前端清除过期 Token。
- 跳转登录页。
- 用户重新登录。

### 11.5 图片无法访问

现象：头像或封面上传成功，但页面无法显示。

处理：

- 检查上传目录是否存在。
- 检查后端是否返回正确 URL。
- 检查 Nginx 是否配置 `/uploads/` 静态访问。
- 检查文件权限。
