# 博客网站部署说明

## 1. 部署架构

前后端分离部署：

```
┌─────────────┐      ┌────────────────┐      ┌──────────┐
│  浏览器      │ ──→  │  Nginx (80/443) │ ──→  │ 后端 API │
│ (Vue 3 SPA) │      │  / → dist/     │      │ :8080    │
└─────────────┘      │  /api/ → proxy  │      └────┬─────┘
                     │  /uploads/ →    │           │
                     │    alias        │      ┌────▼─────┐
                     └────────────────┘      │  MySQL    │
                                             │  :3306    │
                                             ├──────────┤
                                             │  Redis    │
                                             │  :6379    │
                                             └──────────┘
```

生产环境建议使用 Nginx 托管前端静态资源，反向代理 `/api/` 到后端，直接通过文件系统提供 `/uploads/` 访问。

## 2. 环境要求

| 依赖 | 版本 | 说明 |
|------|------|------|
| JDK | 8+ | 运行 Spring Boot 2.7.18 |
| Maven | 3.6+ | 后端依赖管理和打包 |
| Node.js | 18+ | 前端开发和构建 |
| npm | 9+ | 前端包管理 |
| MySQL | 8.0+ | 数据库，utf8mb4 字符集 |
| Redis | 6.0+ | 缓存、限流、分布式锁、RTopic 广播 |
| Nginx | 1.20+ | 生产环境前端托管和反向代理（可选） |

## 3. 数据库初始化

### 3.1 自动初始化

后端配置了 `spring.datasource` 的 `createDatabaseIfNotExist=true`，启动时自动创建 `my_blog` 数据库并执行 `schema.sql` 建表和 `initdata.sql` 插入种子数据。

### 3.2 手动初始化（生产环境）

```sql
CREATE DATABASE my_blog DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

CREATE USER 'my_blog'@'%' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON my_blog.* TO 'my_blog'@'%';
FLUSH PRIVILEGES;
```

按顺序执行初始化脚本：

1. `backend/src/main/resources/db/schema.sql` — 建库 + 建表（35 张业务表）
2. `backend/src/main/resources/db/growth/growth-schema.sql` — 成长模块建表（约 28 张表）+ 种子数据
3. `backend/src/main/resources/db/initdata.sql` — 测试用户数据（201 个用户）

## 4. 后端部署

### 4.1 配置

修改 `backend/src/main/resources/application.yml` 或通过环境变量覆盖：

```properties
# MySQL
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_DATABASE=my_blog
MYSQL_USERNAME=root
MYSQL_PASSWORD=123456

# Redis
REDIS_HOST=localhost
REDIS_PORT=6379
# REDIS_PASSWORD=your_password

# JWT (生产环境必须修改)
JWT_SECRET=replace_with_a_long_random_secret_string_here
JWT_EXPIRE_SECONDS=86400

# 上传目录
UPLOAD_DIR=/data/my-blog/uploads

# COS 云存储（可选，不配置则使用本地/代理上传）
# COS_ENDPOINT=https://cos.ap-guangzhou.myqcloud.com
# COS_REGION=ap-guangzhou
# COS_BUCKET=my-blog-xxxx
# COS_ACCESS_KEY=xxxx
# COS_SECRET_KEY=xxxx

# 邮件（QQ SMTP）
MAIL_HOST=smtp.qq.com
MAIL_PORT=465
MAIL_USERNAME=your@qq.com
MAIL_PASSWORD=your_smtp_authorization_code
```

### 4.2 打包

```bash
cd backend
mvn clean package -DskipTests
```

产物：`backend/target/my-blog.jar`

### 4.3 启动

```bash
java -jar target/my-blog.jar
# 或使用外部配置
java -jar target/my-blog.jar --spring.config.location=file:/etc/my-blog/application.yml
```

生产环境建议通过 systemd、Docker 或进程管理工具（如 PM2、Supervisor）托管。

### 4.4 健康检查

```
GET /api/health
```

返回 `{"code":0,"message":"success","data":{"status":"UP"}}`。后端的 Actuator 健康检查覆盖应用状态、MySQL 连接和 Redis 连接。

## 5. 前端部署

### 5.1 构建

```bash
cd frontend
npm install

```

产物：`frontend/dist/` 目录。

### 5.2 部署方式

将 `dist/` 目录内容复制到 Nginx 静态目录（如 `/var/www/my-blog/dist`）。

## 6. Nginx 配置示例

```nginx
server {
    listen 80;
    server_name example.com;
    client_max_body_size 22m;

    root /var/www/my-blog/dist;
    index index.html;

    # SPA 路由 fallback
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Authorization $http_authorization;

        # SSE 支持
        proxy_http_version 1.1;
        proxy_set_header Connection "";
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 86400s;
        proxy_send_timeout 86400s;
        gzip off;
    }

    # 上传文件直接访问
    location /uploads/ {
        alias /data/my-blog/uploads/;
        expires 7d;
        add_header Cache-Control "public, immutable";
    }
}
```

### SSE 端点说明

后端提供两个 SSE 端点用于实时消息推送：

- `GET /api/notifications/stream` — 系统通知实时推送
- `GET /api/messages/stream` — 私信实时推送

Nginx 配置中 `proxy_buffering off`、`proxy_read_timeout 86400s` 和关闭压缩是 SSE 正常工作所必需的。

## 7. 生产部署步骤

1. 安装 JDK 8+、Node.js 18+、MySQL 8.0+、Redis 6.0+、Nginx 1.20+。
2. 创建 MySQL 数据库和用户。
3. 执行 `schema.sql`、`growth-schema.sql`、`initdata.sql`。
4. 修改后端 `application.yml` 中的数据库、Redis、JWT、邮件配置。
5. 打包后端：`mvn clean package -DskipTests`。
6. 启动后端：`java -jar target/my-blog.jar`。
7. 构建前端：`npm run build`。
8. 将 `dist/` 复制到 Nginx 静态目录。
9. 配置 Nginx 反向代理和上传目录访问。
10. 重启 Nginx。
11. 验证：`GET /api/health` → 200，页面可正常访问。

## 8. 生产安全建议

- JWT 密钥使用最少 32 字符随机字符串，通过环境变量注入。
- 数据库密码、邮件密码、云存储密钥均通过环境变量配置，不提交仓库。
- 启用 Nginx HTTPS（Let's Encrypt 免费证书）。
- 限制管理后台访问 IP 白名单（可选）。
- 定期轮换 JWT 密钥。
- 监控日志文件 `logs/application.log`，配置日志轮转（默认 30 天）。
- 上传目录配置定期清理未使用的临时文件。

## 9. 常见问题

### 9.1 端口被占用

- 后端端口：修改 `server.port` 或设置 `SERVER_PORT` 环境变量。
- 前端端口：修改 `vite.config.js` 的 `server.port`。

### 9.2 数据库连接失败

- 检查 MySQL 是否启动。
- 检查 `application.yml` 中的地址、端口、用户名和密码。
- 检查 `createDatabaseIfNotExist=true` 是否生效（数据库账号需有建库权限）。
- 如应用账号无建库权限，手动创建 `my_blog` 数据库。

### 9.3 跨域问题

- 开发环境：前端 Vite 代理 `/api` 到 `http://localhost:8080`，无跨域。
- 生产环境：Nginx 同域代理，无跨域。
- 后端 CORS 配置允许所有来源（`allowedOrigins: "*"`），仅开发环境生效。

### 9.4 JWT 过期

- 默认 24 小时过期，前端检测到 401 后清除本地 Token 并提示重新登录。
- 可在 `my-blog.jwt.expire-seconds` 调整过期时间。

### 9.5 图片无法访问

- 检查上传目录权限（后端进程需读写权限）。
- 检查 Nginx `alias` 路径是否正确。
- 检查上传返回的 URL 是否为可公开访问的路径。
- COS 模式：检查云存储桶权限和预签名 URL 有效期。

### 9.6 邮件发送失败

- 检查 SMTP 地址、端口和授权码。
- QQ 邮箱需使用授权码而非登录密码。
- 检查防火墙是否放行 SMTP 端口（465/587）。

### 9.7 Redis 连接失败

- 检查 Redis 是否启动。
- 检查 `application.yml` 中的 Redis 地址和密码。
- 生产环境建议使用密码保护 Redis 实例。
