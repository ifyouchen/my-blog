# my-blog 产品说明书

> 版本：v0.1.0  
> 定位：面向技术创作者和技术读者的博客社区平台  
> 最后更新：2026-05-25

---

## 一、项目概述

my-blog 是一个面向技术创作者和技术读者的博客社区 MVP，具备完整的文章发布、阅读、互动、管理后台以及用户成长体系。后端采用 DDD（领域驱动设计）四层架构，前端采用 Vue 3 + Vite，前后端分离部署。

### 核心目标

- 为技术创作者提供流畅的写作与发布体验
- 为技术读者提供高质量内容发现与互动渠道
- 构建基础的用户激励与成长体系

## 八、关键业务流程

### 8.1 文章生命周期

```
┌───────┐     发布     ┌──────────┐     下线     ┌─────────┐
│ DRAFT │ ───────────→  │ PUBLISHED │ ──────────→  │ OFFLINE │
└───────┘               └──────────┘               └─────────┘
    ↑                        │                         │
    │                        │ 重新发布                 │
    └────────────────────────┘                         │
                                                       │
    删除 ←───────── DELETED ←──────────────────────────┘
```

- DRAFT → PUBLISHED：记录 `published_at`
- 所有状态变更在 `Article` 聚合根中校验合法性

### 8.2 图片上传流程

```
 前端                        后端                       COS
  │                          │                         │
  │  POST /uploads/presigned │                         │
  │ ────────────────────────→│                         │
  │   ← { presignedUrl }     │ 生成预签名 URL           │
  │                          │                         │
  │  PUT file → presignedUrl │                         │
  │ ──────────────────────────────────────────────────→│
  │                          │                         │
  │  ← { url, thumbnailUrl } │                         │
```

- 前端**不接触**任何云存储密钥
- COS 未配置时降级为后端代理上传（`POST /api/uploads/images`，后端转发存储）
- 上传限流：图片 10 次/60s/IP，文件 5 次/60s/IP
- 支持三种 scope：cover、avatar、chat

### 8.3 缓存策略

- **两级缓存**：Caffeine（L1） + Redis（L2）
- **序列化**：GenericJackson2JsonRedisSerializer
- **跨实例失效**：Redisson RTopic 发布订阅
- **启动清理**：`@PostConstruct clearOldCacheData`

### 8.4 认证流程

```
  Client                  Server
   │                        │
   │  POST /api/auth/login  │
   │ ──────────────────────→│
   │                        │ 验证密码（BCrypt）
   │                        │ 生成 JWT（HS256, userId/username/role）
   │  ← { token, user }     │
   │                        │
   │  GET /api/articles     │
   │  Authorization: Bearer │
   │ ──────────────────────→│
   │                        │ JwtAuthenticationInterceptor 解析令牌
   │                        │ 验证用户状态（即使 JWT 有效）
```

### 8.5 成长体系

```
用户行为（发文、评论、点赞、签到等）
       │
       ▼
  事件触发 ───→ GrowthRuleConfig 匹配
       │
       ├──→ 经验值变更（user_growth_account）
       │      └──→ 等级升级检测 → 等级特权解锁 + 奖励发放
       │
       └──→ 积分变更（user_point_account）
              └──→ 可消费：文章解锁、推荐申请等
```

## 九、安全设计

- **密码存储**：BCrypt 哈希
- **JWT 认证**：HMAC-SHA256 签名，24h 过期
- **文件上传**：限制 20MB/文件，22MB/请求，频率限制
- **XSS 防护**：DOMPurify 过滤用户输入的 HTML
- **SQL 注入**：MyBatis 参数绑定
- **软删除**：所有数据不可物理删除
- **操作审计**：管理员所有操作记入 `admin_log`
- **敏感词过滤**：WARN（警告）和 BLOCK（拦截）两级
- **验证码**：邮件验证码限制（IP + 邮箱双重频率）
- **接口限流**：每 IP 每分钟 20 次默认限制
