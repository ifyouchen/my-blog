# my-blog 部署注意事项

> 开源部署指南  
> 适用版本：v0.1.0

---

## 一、环境要求

| 依赖 | 最低版本 | 备注 |
|------|----------|------|
| JDK | 1.8 (8) | 不支持 Java 11+（当前项目使用 JDK 8 API） |
| Maven | 3.6+ | 构建后端 |
| Node.js | 18+ | 构建前端 |
| MySQL | 8.0+ | 需要 `utf8mb4` 支持 |
| Redis | 6.0+ | 缓存、分布式锁、RTopic |
| Nginx | 推荐 1.20+ | 生产环境反向代理 |

## 二、快速启动（开发环境）

### 2.1 克隆代码

```bash
git clone <仓库地址>
cd my-blog
```

### 2.2 启动后端

```bash
cd backend
mvn spring-boot:run
```

- 服务默认监听 `http://localhost:8080`
- 首次启动会自动创建 `my_blog` 数据库并执行建表脚本和种子数据

### 2.3 启动前端

```bash
cd frontend
npm install
npm run dev
```

- 服务默认监听 `http://localhost:5173`
- Vite 代理将 `/api` 请求转发到 `http://localhost:8080`

## 三、⚠️ 重要：密钥与凭据配置（开源前必读）

本项目提交到仓库的 `application.yml` 中包含**某些真实凭据的默认值**。如果您 Fork 或克隆此项目，请**立即执行以下操作**：

### 3.1 需要替换的凭据

| 配置项 | 位置 | 说明 | 操作建议 |
|--------|------|------|----------|
| **COS SecretId** | `my-blog.cos.secret-id` | 腾讯云对象存储密钥 ID | 替换为你的值或留空 |
| **COS SecretKey** | `my-blog.cos.secret-key` | 腾讯云对象存储密钥 Key | 替换为你的值或留空 |
| **QQ 邮箱 SMTP 密码** | `spring.mail.password` | 邮箱登录授权码 | 替换为你的值或留空 |
| **JWT Secret** | `my-blog.jwt.secret` | JWT 签名密钥 | 替换为随机字符串（生产环境务必修改） |

### 3.2 通过环境变量覆盖

所有敏感配置均支持通过环境变量覆盖（`${VAR_NAME:default}` 模式）。推荐在部署时使用环境变量而非直接修改文件：

```bash
# 必须配置（无合理默认值）
export COS_SECRET_ID=your-cos-secret-id
export COS_SECRET_KEY=your-cos-secret-key
export MAIL_PASSWORD=your-smtp-auth-code
export JWT_SECRET=your-strong-random-secret   # 目前必须直接改 yml

# 可选配置（有合理默认值）
export MYSQL_HOST=localhost
export MYSQL_PORT=3306
export MYSQL_DATABASE=my_blog
export MYSQL_USERNAME=root
export MYSQL_PASSWORD=123456
export REDIS_HOST=127.0.0.1
export REDIS_PORT=6379
export REDIS_PASSWORD=
export FRONTEND_BASE_URL=https://your-domain.com
export MAIL_HOST=smtp.qq.com
export MAIL_PORT=465
export MAIL_USERNAME=your-email@qq.com
```

> **注意**：`my-blog.jwt.secret` 当前未配置环境变量兜底，需要直接编辑 `application.yml` 或在启动参数中传入：  
> `--my-blog.jwt.secret=your-secret`

### 3.3 推荐做法

```
1. 复制一份配置模板：
   cp backend/src/main/resources/application.yml backend/src/main/resources/application.yml.example

2. 将 application.yml.example 中的所有真实密钥替换为占位符：
   secret-id: ${COS_SECRET_ID:your-cos-secret-id}
   password: ${MAIL_PASSWORD:your-smtp-password}

3. 将 application.yml 加入 .gitignore 或将其中的默认值清空

4. 通过环境变量或外部配置文件传入真实凭据
```

## 四、COS 云存储配置

### 4.1 腾讯云 COS

1. 在腾讯云控制台创建存储桶（Bucket）
2. 配置 `application.yml`：

```yaml
my-blog:
  cos:
    secret-id: ${COS_SECRET_ID}
    secret-key: ${COS_SECRET_KEY}
    region: ap-shanghai          # 你的存储桶地域
    bucket: myblog-1322876242    # 你的存储桶名称
    domain: https://your-cdn-domain.com  # 可选：CDN 加速域名
```

3. 存储桶需设置**公有读私有写**权限
4. CORS 配置允许前端域名

### 4.2 不使用 COS 的本地模式

如果未配置 COS，系统会自动检测并使用**后端代理上传**模式（上传到本地 `uploads/` 目录）。

```yaml
my-blog:
  cos:
    secret-id: ""   # 留空或取消配置
```

上传的文件会保存在 `backend/uploads/`，通过 `http://localhost:8080/api/uploads/files/...` 访问。生产环境需配置 Nginx 直接托管此目录。

## 五、邮件服务配置

### 5.1 QQ 邮箱

```yaml
spring:
  mail:
    host: smtp.qq.com
    port: 465
    username: your-email@qq.com
    password: your-smtp-auth-code    # 非 QQ 密码，是 SMTP 授权码
    properties:
      mail.smtp.ssl.enable: true
```

SMTP 授权码在 QQ 邮箱 → 设置 → 账户 → POP3/SMTP 服务中生成。

### 5.2 163 邮箱

```yaml
spring:
  mail:
    host: smtp.163.com
    port: 465
    username: your-email@163.com
    password: your-smtp-auth-code
```

### 5.3 Gmail

```yaml
spring:
  mail:
    host: smtp.gmail.com
    port: 587
    username: your-email@gmail.com
    password: your-app-password
    properties:
      mail.smtp.starttls.enable: true
```

## 六、生产部署

### 6.1 后端打包部署

```bash
cd backend
mvn clean package -DskipTests
# 生成 target/my-blog-backend-0.1.0.jar

java -jar target/my-blog-backend-0.1.0.jar \
  --spring.profiles.active=prod \
  --my-blog.jwt.secret=your-strong-secret \
  --COS_SECRET_ID=xxx \
  --COS_SECRET_KEY=xxx
```

### 6.2 前端打包

```bash
cd frontend
npm install
npm run build
# 生成 frontend/dist/
```

### 6.3 Nginx 配置示例

```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    # 前端静态资源
    root /path/to/frontend/dist;
    index index.html;

    # SPA 路由
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # SSE 支持
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 86400s;
    }

    # 上传文件（本地模式）
    location /api/uploads/files/ {
        alias /path/to/backend/uploads/;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml text/javascript image/svg+xml;
    gzip_min_length 1024;
}
```

### 6.4 使用 systemd 管理后端服务

```ini
[Unit]
Description=my-blog Backend
After=network.target mysql.service redis.service

[Service]
Type=simple
User=deploy
WorkingDirectory=/opt/my-blog/backend
ExecStart=/usr/bin/java -jar /opt/my-blog/backend/target/my-blog-backend-0.1.0.jar
Environment=COS_SECRET_ID=xxx
Environment=COS_SECRET_KEY=xxx
Environment=MAIL_PASSWORD=xxx
Environment=SPRING_SQL_INIT_MODE=never
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

## 七、数据库初始化

### 7.1 自动建表

后端启动时可通过配置自动执行 SQL 脚本：

```yaml
spring:
  sql:
    init:
      mode: always    # 首次启动设为 always，之后改为 never
      schema-locations:
        - classpath:db/schema.sql
        - classpath:db/growth/growth-schema.sql
      data-locations:
        - classpath:db/initdata.sql
        - classpath*:db/articles/*.sql
        - classpath:db/category_groups_migration.sql
```

### 7.2 种子数据说明

| 脚本 | 内容 | 是否必须 |
|------|------|----------|
| `schema.sql` | 核心表结构 | 是 |
| `growth/growth-schema.sql` | 成长模块表结构 | 是 |
| `initdata.sql` | 基础数据（管理员账号、分类组、成长规则、徽章定义等） | 是 |
| `articles/*.sql` | 示例文章（技术教程内容） | 可选 |
| `category_groups_migration.sql` | 分类组数据迁移 | 必要时使用 |

> 注意：`articles/*.sql` 中的文章来自公开技术教程，仅供演示。生产环境可删除这些脚本。

### 7.3 手动初始化

```bash
mysql -u root -p my_blog < backend/src/main/resources/db/schema.sql
mysql -u root -p my_blog < backend/src/main/resources/db/growth/growth-schema.sql
mysql -u root -p my_blog < backend/src/main/resources/db/initdata.sql
```

## 八、安全建议

### 8.1 开源前安全检查清单

- [ ] 移除或替换 `application.yml` 中的所有真实凭据（COS、SMTP）
- [ ] 检查 `db/articles/*.sql` 是否包含敏感或版权内容
- [ ] 检查 `git log` 中是否有历史提交泄露过密钥（如果已推送过，需轮换密钥）
- [ ] 将 `application.yml` 加入 `.gitignore`，创建 `application.yml.example`
- [ ] 确保 `.gitignore` 不遗漏 `logs/`、`uploads/`、`target/`、`dist/`
- [ ] JWT 密钥必须使用强随机字符串（至少 32 字符）

### 8.2 生产环境安全

- **HTTPS**：全站强制 HTTPS，配置 SSL 证书
- **密钥轮换**：定期更换 COS 密钥、SMTP 授权码、JWT 密钥
- **数据库**：不使用 root 账号连接，创建专用数据库用户并限制权限
- **Redis**：设置密码（`spring.redis.password`），绑定内网 IP
- **限流**：`my-blog.rate-limit.ip-max-requests-per-minute` 根据实际情况调整
- **上传限制**：配置合理的上传大小和频率限制
- **防火墙**：仅开放 80/443 端口，后端端口（8080）仅允许内网访问

### 8.3 JWT 安全

- 生产环境 `expire-seconds` 建议设为 `86400`（24 小时）或更短
- JWT 密钥建议 64 字节以上的随机字符串
- 可考虑添加 refresh token 机制（当前版本未实现）

## 九、常见问题

### Q1: 启动报错 "数据库连接失败"

检查 MySQL 是否运行，连接参数是否正确：
```bash
# 默认连接参数
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_DATABASE=my_blog
MYSQL_USERNAME=root
MYSQL_PASSWORD=123456
```

### Q2: 启动报错 "Redis 连接失败"

检查 Redis 是否运行：
```bash
redis-cli ping  # 应返回 PONG
```
如果 Redis 设置了密码，通过环境变量传入：`REDIS_PASSWORD=xxx`

### Q3: 前端页面打不开，接口 404

确认前端是否运行在 5173 端口，Vite 代理是否指向正确的后端地址：
```javascript
// frontend/vite.config.js
server: {
    proxy: {
        '/api': {
            target: 'http://localhost:8080',  // 确认后端端口
            changeOrigin: true
        }
    }
}
```

### Q4: 图片上传失败

- **COS 模式**：检查 COS 配置和存储桶权限（需公有读）
- **本地模式**：检查 `uploads/` 目录是否可写，Nginx 是否代理了 `/api/uploads/files/`
- **上传限流**：默认每 IP 每 60 秒 10 次，可通过环境变量调整

### Q5: 邮件发送失败

- 检查 SMTP 配置（特别是授权码，非邮箱密码）
- QQ 邮箱需使用 SSL 端口 465
- 检查是否被邮件服务商限流（默认每邮箱每 30 秒最多一封）

### Q6: 文章发布的文章显示为英文乱码

- 检查 MySQL 连接 URL 中的 `characterEncoding=utf8` 参数
- 检查数据库和表是否为 `utf8mb4` 字符集

### Q7: 怎么创建管理员账号？

默认种子数据中已创建管理员账号：
- 邮箱：`admin@myblog.com`
- 密码：`admin123`

**生产环境请立即修改默认管理员密码。**
