## 1. 管理后台权限前置 — 已完成

### 改动内容

**新增 `AccessDeniedView.vue`：**

- 通用 403 页面，展示"无权限访问"提示。
- 提供"返回首页"按钮和"切换账号"按钮。
- "切换账号"执行 `logout()` 后跳转 `/login`。

**`router/index.js`：**

- 新增 `/403` 路由，`name: 'accessDenied'`，`meta.title = '无权限访问'`。
- `/admin` 父路由增加 `meta.requiresAdmin = true`，保留 `requiresAuth = true`。
- 路由守卫逻辑调整为：
  1. 先检查 `requiresAuth`，未登录弹登录框。
  2. 再检查 `requiresAdmin`，已登录但 `role !== 'ADMIN'` 时跳转 `/403?from=原路径`。
- 子路由不重复写 `requiresAdmin`，依赖 `to.matched.some()` 检查。

### 验收要点

- 未登录访问 `/admin`，应回首页并弹登录框。
- 普通登录用户访问 `/admin`，应进入 `/403?from=/admin`。
- ADMIN 用户访问 `/admin` 正常进入。

## 2. 移动搜索体验精修 — 已完成

### 改动内容

**`SiteHeader.vue`：**

- 引入 `nextTick`。
- 新增 `mobileSearchInputRef = ref(null)`。
- `openMobileSearch()` 打开面板后 `nextTick(() => mobileSearchInputRef.value?.focus())` 自动聚焦。
- 移除搜索输入框的 `autofocus`，改用 `ref="mobileSearchInputRef"`。
- 新增 `handleSearchKeydown(event)`：Escape 时关闭搜索面板。
- `watch(mobileSearchOpen)`：
  - 打开时给 `document.body` 加 `mobile-search-open` class，绑定 keydown 监听。
  - 关闭时移除 body class 和 keydown 监听。
- `watch(() => route.fullPath)`：
  - 路由变化时关闭移动搜索、移动菜单、通知弹层、用户菜单。
  - 当 `route.path === '/search'` 时，将 `route.query.keyword` 回填到 `keyword`。
- `onUnmounted` 中清理 body class 和 keydown 监听器。

**`global.css`：**

- 新增 `body.mobile-search-open { overflow: hidden; }`，防止搜索面板打开时背景滚动。

### 验收要点

- 390px 首页点击搜索图标，输入框自动聚焦。
- 按 Escape 关闭搜索面板。
- 输入 `Redis` 搜索后进入 `/search?keyword=Redis`，面板关闭。
- 在搜索页刷新或返回时，Header 搜索框能回填 `Redis`。

## 3. 广告登录态同步 — 已完成

### 改动内容

**`AdBanner.vue`：**

- 用 `useSession()` 替换 `getToken()` 判断。
- `const { state, isLoggedIn, initializeSession } = useSession()`。
- `onMounted` 中先 `await initializeSession()`，再并发执行 `fetchAds()` 和 `fetchDismissedIds()`。
- `fetchDismissedIds()`：未登录时清空 `serverDismissedIds` 并返回，不请求 `/ads/dismissed-ids`。
- `dismissSession()` 和 `dismissToday()`：只有 `isLoggedIn.value` 为 true 时才调用 `dismissAdApi()`。
- 新增 `watch(isLoggedIn)`：
  - 登录后重新拉取 dismissed ids。
  - 退出后清空 `serverDismissedIds`。

### 验收要点

- 游客访问首页/文章详情，不请求 `/ads/dismissed-ids`。
- 游客关闭广告，不请求 `/ads/{id}/dismiss`，本地隐藏生效。
- 登录后访问广告位，会请求 dismissed ids。
- 登录后关闭广告，会调用服务端 dismiss。

## 4. 全站 401/403 统一反馈 — 已完成

### 改动内容

**`http.js`：**

- 新增 `dispatchWithCooldown(eventName, detail, cooldownMs = 1500)` 函数，1500ms 内同类型事件只派发一次。
- `createRequestError` 中：
  - 401 时清 session 并派发 `my-blog-auth-required` 事件。
  - 403 时派发 `my-blog-forbidden` 事件。

**`App.vue`：**

- 引入 `useRouter`。
- 监听 `my-blog-auth-required`：弹登录框，登录成功后 `router.replace(router.currentRoute.value.fullPath)`。
- 监听 `my-blog-forbidden`：`toast.error(message)`；如果当前路径以 `/admin` 开头，跳转 `/403?from=currentFullPath`。
- `onUnmounted` 清理两个全局事件监听器。
- 保持已有 `my-blog-auth-expired` 由 session store 清理登录态。

### 验收要点

- 过期 token 触发 401 时清 session，并弹登录框。
- 普通用户触发后台 403 时显示 toast，并跳转 403 页面。
- 并发多个 401/403 请求时，不连续刷屏弹窗或 toast（1500ms 冷却）。
