# my-blog 技术架构

## 1. 架构总览

采用前后端分离架构，后端 DDD 四层架构，前端 Vue 3 + Vite。

```
浏览器
   │
   │ HTTP / JSON / JWT
   ▼
Vue 3 前端应用 (Vite, port 5173)
   │
   │ REST API (proxy /api → localhost:8080)
   ▼
Spring Boot 后端服务 (port 8080)
   │
   ├── MySQL 8.0+ (主数据库)
   ├── Redis 6+ (缓存 + 分布式锁 + SSE)
   └── COS S3 (云存储，可选)
```

生产环境：Nginx → Vue 静态资源 + 反向代理 /api → Spring Boot

## 4. 后端分层架构（DDD）

```
Interfaces 层                   应用入口
  ├── Controller                REST 端点（无业务逻辑）
  ├── DTO (request/response)    入参/出参
  ├── Mapper (MapStruct)        DTO ↔ Command/Query 转换
  ├── Interceptor               拦截器（鉴权、限流）
  ├── AOP                       日志切面
  └── GlobalExceptionHandler    统一异常处理
        │
        ▼
Application 层                 用例编排
  ├── Service                   应用服务（@Transactional）
  ├── Command / Query           写/读命令对象
  ├── Event / Listener          领域事件/监听器
  ├── Assembler                 DTO ↔ 领域对象
  └── Cache                     缓存
        │
        ▼
Domain 层                      核心业务
  ├── Aggregate                 聚合根（充血模型）
  ├── ValueObject               值对象（不可变）
  ├── Service                   领域服务
  ├── Repository                仓储接口
  └── Event                     领域事件
        │
        ▼
Infrastructure 层              技术实现
  ├── Repository (MyBatis)      仓储实现
  ├── Entity (DO)               持久化对象
  ├── Mapper (XML)              MyBatis XML
  ├── Security                  JWT + AuthContext
  ├── Mail                      邮件发送（SMTP）
  ├── Storage                   文件存储（Local / COS）
  ├── Config                    配置
  └── Logging                   日志
```

**依赖方向**：`Interfaces → Application → Domain`，`Infrastructure → Domain`

**禁止**：
- Domain 依赖 Interfaces 或 Infrastructure
- Controller 直接调用 Mapper
- Controller 包含业务逻辑

## 6. 前端架构

```
frontend/src/
├── api/           (26 modules) 基于 fetch() 的 HTTP 客户端
├── components/    (37 reusable components)
├── composables/   (7 composables: toast, confirm, login modal, etc.)
├── stores/        (2 reactive stores: session.js, growth.js)
├── views/         (26 page views)
├── router/        (Vue Router config, 40+ routes)
├── utils/         (10 utility modules)
├── constants/     (app constants)
└── styles/        (global CSS)
```

**特点**：
- 无 Pinia 依赖，使用 `reactive()` 单例模式管理状态
- 使用原生 `fetch()` 而非 Axios
- 状态管理通过 `session.js`（localStorage 持久化）+ `growth.js`（内存）
- SSE 实现实时通知和消息推送
- TipTap 富文本编辑器实现文章编写
- 构建产物通过 rollup 分块优化（editor-core, admin, markdown, jszip 等独立 chunk）

## 7. 缓存架构（两级缓存）

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  Instance 1  │    │  Instance 2  │    │  Instance 3  │
│              │    │              │    │              │
│  Caffeine L1 │    │  Caffeine L1 │    │  Caffeine L1 │
│      │       │    │      │       │    │      │       │
│  Redis L2    │    │  Redis L2    │    │  Redis L2    │
└──────┼───────┘    └──────┼───────┘    └──────┼───────┘
       │                   │                   │
       └─────────── Redis Cluster ─────────────┘
                   (RTopic 缓存失效广播)
```

**实现**：`TwoTierCache`（`application/cache/TwoTierCache.java`）
- L1：Caffeine 本地缓存（毫秒级访问）
- L2：StringRedisTemplate + GenericJackson2JsonRedisSerializer
- 失效广播：Redisson RTopic（Publish/Subscribe）

## 8. 认证流程

```
用户提交账号密码
  → 后端校验密码 (BCrypt)
  → 校验用户状态 (NORMAL)
  → 生成 JWT (HS256, 含 userId/username/role, 24h 过期)
  → 前端保存 Token 到 localStorage
  → 后续请求携带 Authorization: Bearer <token>
  → JwtAuthenticationInterceptor 解析 Token
  → 设置 AuthContext (ThreadLocal)
  → 即使 JWT 有效，仍需校验用户当前状态
  → 响应后清理 AuthContext
```

## 9. SSE 实时推送

两个 SSE 端点：
- `/api/notifications/stream` — 通知未读计数实时推送
- `/api/messages/stream` — 消息未读计数实时推送

实现方式：Spring `SseEmitter` + Redis pub/sub 跨实例广播

## 10. 文件上传架构

两种上传策略：

**1. COS 预签名 URL（生产推荐）**
```
前端请求 POST /api/uploads/presigned-url → 后端生成预签名 URL
前端直接 PUT 文件到 COS（不经过后端）
前端使用返回的 URL 作为图片地址
```

**2. 后端代理上传（开发降级）**
```
前端 POST /api/uploads/images (multipart/form-data)
后端接收文件 → 校验 → 保存到本地目录 → 返回 URL
```

## 11. 跨模块通信（成长模块）

```
博客核心 (com.myblog.*)          成长模块 (com.myblog.growth.*)
    │                                   │
    │  发布 Spring 事件                  │
    │  (ArticleLikedEvent,              │
    │   ArticleViewedEvent,              │  @EventListener 订阅
    │   CommentCreatedEvent,             │  处理经验/积分发放
    │   ArticleFavoritedEvent,           │
    │   UserFollowedEvent,              │
    │   ArticlePublishedEvent)           │
    │                                   │
    └─────────── ApplicationEventPublisher ──────────→
```

博客侧只发事件，不感知成长模块；成长模块只监听，不修改博客代码。

## 12. 数据流示例：文章解锁

```
用户点击解锁
  → POST /api/articles/{id}/unlock
  → ArticleUnlockController
  → ArticleUnlockAppService.unlock() [@Transactional]
      ├── 查 blog_article (need_unlock, price, author_id)
      ├── 查是否已解锁 (article_unlock_relation)
      ├── 查积分余额 (user_point_account)
      ├── INSERT article_unlock_order
      ├── PointAppService.deductPoints() → user_point_account
      ├── INSERT article_unlock_relation
      ├── RevenueShareService.createShareRecord()
      │     → revenue_share_journal (50:50 分账)
      └── UPDATE article_unlock_order status=SUCCESS
```
