# 博客网站第一版技术架构

## 1. 架构目标

第一版采用前后端分离架构，保证开发清晰、部署简单、后续扩展方便。

技术栈固定为：

- 前端：Vue 3。
- 后端：JDK 8、Spring Boot 2.7.18。
- 数据库：MySQL。
- 认证：JWT。

后端代码实现必须遵循 [开发规范](10-开发规范.md)，采用 DDD 四层架构和阿里巴巴 Java 开发手册约束。

## 2. 总体架构

```text
浏览器
  |
  | HTTP / JSON / JWT
  v
Vue 3 前端应用
  |
  | REST API
  v
Spring Boot 后端服务
  |
  | SQL
  v
MySQL 数据库
```

生产环境建议：

```text
用户浏览器 -> Nginx -> Vue 静态资源
用户浏览器 -> Nginx 反向代理 -> Spring Boot -> MySQL
```

## 3. 前端技术方案

### 3.1 推荐依赖

- Vue 3：前端框架。
- Vue Router：路由管理。
- Pinia：状态管理。
- Axios：HTTP 请求。
- Markdown 编辑器组件：文章编辑。
- Markdown 渲染组件：文章详情展示。
- 代码高亮库：展示文章中的代码块。

### 3.2 前端目录建议

```text
frontend/
  src/
    api/
    assets/
    components/
    router/
    stores/
    views/
    utils/
```

### 3.3 前端职责

- 页面展示。
- 表单校验。
- 路由守卫。
- Token 存储和请求头注入。
- 调用后端接口。
- Markdown 编辑和预览。

## 4. 后端技术方案

### 4.1 推荐依赖

- Spring Boot 2.7.18：后端基础框架。
- Spring Web：REST API。
- Spring Security：认证和权限控制。
- MyBatis：数据访问，使用 Mapper 接口 + XML，仓储接口仍定义在领域层。
- JWT 工具库：Token 生成和校验。
- MySQL Driver：数据库连接。
- Lombok：减少样板代码，可选。

### 4.2 后端目录建议

```text
backend/
  src/main/java/
    com/myblog/
      interfaces/
      application/
      domain/
      infrastructure/
      shared/
```

具体包结构以 [开发规范](10-开发规范.md) 为准。

### 4.3 后端职责

- 用户认证。
- 权限校验。
- 业务规则处理。
- 数据读写。
- 文件上传。
- 统一异常处理。
- 统一返回格式。
- 管理后台接口。

## 5. 数据库方案

数据库使用 MySQL，核心表包括：

- `user`
- `article`
- `category`
- `tag`
- `article_tag`
- `comment`
- `article_like`
- `article_favorite`
- `article_view`
- `admin_log`

第一版直接使用 MySQL 查询实现文章列表、筛选和搜索。后续如果数据量变大，可以引入 Elasticsearch 或 OpenSearch。

## 6. 认证流程

```text
用户提交账号密码
  -> 后端校验账号密码
  -> 后端校验用户状态
  -> 后端生成 JWT
  -> 前端保存 JWT
  -> 前端请求受保护接口时携带 Authorization 请求头
  -> 后端解析 JWT
  -> 后端查询用户状态和角色
  -> 后端执行权限校验
```

JWT 中建议包含：

- `userId`
- `username`
- `role`
- `issuedAt`
- `expiresAt`

注意：即使 JWT 未过期，后端也需要校验用户当前状态，防止禁用用户继续操作。

## 7. 文章发布流程

```text
用户进入发布页
  -> 前端填写标题、摘要、正文、分类、标签
  -> 前端提交文章数据
  -> 后端校验登录状态
  -> 后端校验字段
  -> 后端保存文章
  -> 后端保存文章标签关联
  -> 返回文章 ID
```

发布为 `PUBLISHED` 状态时写入 `published_at`。保存草稿时状态为 `DRAFT`。

## 8. 文件上传流程

```text
用户选择图片
  -> 前端上传 multipart/form-data
  -> 后端校验文件类型和大小
  -> 后端保存到本地上传目录
  -> 后端返回文件访问 URL
  -> 前端将 URL 写入头像或文章封面字段
```

第一版默认本地存储。生产环境可将上传目录挂载到服务器固定路径，后续可替换为对象存储。

## 9. 跨域方案

开发环境前端和后端端口不同，需要开启 CORS：

- 允许前端开发地址访问后端接口。
- 允许 `Authorization` 请求头。
- 允许常用 HTTP 方法：`GET`、`POST`、`PUT`、`PATCH`、`DELETE`。

生产环境建议通过 Nginx 统一域名和反向代理，减少跨域问题。

## 10. 环境规划

| 环境 | 说明 |
|---|---|
| 本地开发环境 | 前端、后端和数据库在本机运行。 |
| 测试环境 | 部署完整服务，用于联调和验收。 |
| 生产环境 | Nginx + Spring Boot + MySQL。 |

## 11. 后续扩展方向

以下能力作为后续版本扩展：

- Redis 缓存热门文章、登录黑名单、阅读量缓冲。
- 对象存储保存头像、封面和文章图片。
- Elasticsearch 或 OpenSearch 支持全文搜索。
- 消息通知系统。
- 专栏系统。
- 付费内容和会员体系。
- 推荐算法。
- 操作审计和风控策略。
