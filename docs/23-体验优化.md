# 下一批体验优化详细实现步骤

## Summary

基于 `docs/20` 已完成内容，继续做 4 个前端体验收口：管理员权限前置、移动搜索细节、广告登录态同步、全站 401/403 统一反馈。只改前端，不改后端接口、数据库或管理后台业务逻辑。

## Key Changes

### 1. 管理后台权限前置

- 新增 `frontend/src/views/AccessDeniedView.vue`，作为通用 403 页面：
    - 引入 `SiteHeader`。
    - 展示“无权限访问”“当前账号没有访问该页面的权限”。
    - 提供“返回首页”按钮和“切换账号”按钮。
    - “切换账号”执行 `logout()` 后跳转 `/login`。

- 在 `frontend/src/router/index.js`：
    - 新增路由 `/403`，`name: 'accessDenied'`，`meta.title = '无权限访问'`。
    - 给 `/admin` 父路由增加 `meta.requiresAdmin = true`，保留 `requiresAuth = true`。
    - 路由守卫顺序改为：先处理 `requiresAuth`，再处理 `requiresAdmin`。
    - 已登录但 `state.user?.role !== 'ADMIN'` 时跳转：
        - `{ path: '/403', query: { from: to.fullPath }, replace: true }`
    - 不在子路由重复写 `requiresAdmin`，依赖 `to.matched.some(record => record.meta.requiresAdmin)`。

### 2. 移动搜索体验精修

- 在 `frontend/src/components/SiteHeader.vue`：
    - 引入 `nextTick`。
    - 新增 `mobileSearchInputRef = ref(null)`。
    - `openMobileSearch()` 设置 `mobileSearchOpen = true` 后 `nextTick(() => mobileSearchInputRef.value?.focus())`。
    - 移动搜索输入框绑定 `ref="mobileSearchInputRef"`，移除不稳定的 `autofocus`。
    - 新增 `handleSearchKeydown(event)`：`Escape` 时关闭搜索面板。
    - `watch(mobileSearchOpen)`：
        - 打开时给 `document.body` 加 `mobile-search-open`，绑定 `keydown`。
        - 关闭时移除 body class 和 keydown。
    - `onUnmounted` 里确保移除 body class 和 keydown。
    - `watch(() => route.fullPath)`：
        - 路由变化时关闭移动搜索、移动菜单、通知弹层、用户菜单。
        - 当 `route.path === '/search'` 时，将 `route.query.keyword` 回填到 `keyword`。
    - `submitSearch()` 使用 `keyword.value.trim()`，提交后关闭移动搜索和抽屉。

- 在 `frontend/src/styles/global.css`：
    - 新增 `body.mobile-search-open { overflow: hidden; }`。
    - 保持移动搜索面板 `z-index` 高于 header 和抽屉遮罩。
    - 确认 390px 下搜索面板输入框、取消、搜索按钮不换行挤压。

### 3. 广告登录态同步

- 在 `frontend/src/components/AdBanner.vue`：
    - 用 `useSession()` 替换单纯 `getToken()` 判断。
    - 引入 `watch`。
    - `const { state, isLoggedIn, initializeSession } = useSession()`。
    - `onMounted` 中先 `await initializeSession()`，再并发执行 `fetchAds()` 和 `fetchDismissedIds()`。
    - `fetchDismissedIds()`：
        - `!state.ready` 时直接返回或等待初始化。
        - 未登录时 `serverDismissedIds.value = new Set()`，不请求 `/ads/dismissed-ids`。
        - 登录时才调用 `getDismissedAdIdsApi()`。
    - `dismissSession()` 和 `dismissToday()`：
        - 本地隐藏逻辑保留。
        - 只有 `isLoggedIn.value` 为 true 时才调用 `dismissAdApi()`。
    - `watch(isLoggedIn)`：
        - 登录后重新拉取 dismissed ids。
        - 退出后清空 `serverDismissedIds`。
    - 曝光/点击接口继续静默容错，不改变现有行为。

### 4. 全站 401/403 统一反馈

- 在 `frontend/src/api/http.js`：
    - 401 时保留现有 `clearSession()`。
    - 额外派发：
        - `window.dispatchEvent(new CustomEvent('my-blog-auth-required', { detail: { message } }))`
    - 403 或业务错误 `code === 403` 时派发：
        - `window.dispatchEvent(new CustomEvent('my-blog-forbidden', { detail: { message } }))`
    - 增加简单冷却，避免同一批并发请求连续触发多个全局事件；建议 1500ms 内只派发一次同类事件。

- 在 `frontend/src/App.vue`：
    - 引入 `useRouter`。
    - 监听 `my-blog-auth-required`：
        - 弹登录框，文案使用“登录后继续操作”。
        - 登录成功后 `router.replace(router.currentRoute.value.fullPath)`。
    - 监听 `my-blog-forbidden`：
        - `toast.error(message || '无权限访问')`。
        - 如果当前路径以 `/admin` 开头，跳转 `/403?from=currentFullPath`。
    - `onUnmounted` 清理两个全局事件监听器。
    - 保持已有 `my-blog-auth-expired` 由 session store 清理登录态。

## Test Plan

- 执行 `cd frontend && npm run build`。
- 权限：
    - 未登录访问 `/admin`、`/admin/users`，应回首页并弹登录框。
    - 普通登录用户访问 `/admin`，应进入 `/403?from=/admin`，不展示后台主体。
    - ADMIN 用户访问 `/admin` 正常进入。
- 移动搜索：
    - 390px 首页点击搜索图标，输入框自动聚焦。
    - 按 `Escape` 关闭搜索面板。
    - 输入 `Redis` 搜索后进入 `/search?keyword=Redis`，面板关闭。
    - 在搜索页刷新或返回时，Header 搜索框能回填 `Redis`。
- 广告：
    - 游客访问首页/文章详情，不请求 `/ads/dismissed-ids`。
    - 游客关闭广告，不请求 `/ads/{id}/dismiss`，本地隐藏生效。
    - 登录后访问广告位，会请求 dismissed ids。
    - 登录后关闭广告，会调用服务端 dismiss。
- 全局错误：
    - 过期 token 触发 401 时清 session，并弹登录框。
    - 普通用户触发后台 403 时显示 toast，并跳转 403 页面。
    - 并发多个 401/403 请求时，不连续刷屏弹窗或 toast。

## Assumptions

- 本轮只改前端。
- 管理员权限前置只做体验保护，后端权限仍是最终兜底。
- 新增 `/403` 页面是通用无权限页，不替代登录弹窗。
- 广告曝光/点击接口仍允许失败静默，不把统计失败暴露给用户。
