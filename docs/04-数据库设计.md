# 博客网站第一版数据库设计

## 1. 设计说明

数据库使用 MySQL。第一版采用逻辑删除策略，避免物理删除影响统计和关联数据。数据库实现必须遵循 [开发规范](10-开发规范.md)。

所有业务表必须包含以下通用字段：

- `id`：主键，`BIGINT UNSIGNED`，自增或雪花 ID。
- `created_at`：创建时间。
- `updated_at`：更新时间。
- `deleted_at`：删除时间，`NULL` 表示未删除。
- `version`：乐观锁版本号。

软删除统一使用 `deleted_at IS NULL` 判断，禁止物理删除。

## 13. 关系说明

- `user.id` 与 `article.user_id` 为一对多关系。
- `category.id` 与 `article.category_id` 为一对多关系。
- `article` 与 `tag` 通过 `article_tag` 建立多对多关系。
- `article.id` 与 `comment.article_id` 为一对多关系。
- `user.id` 与 `comment.user_id` 为一对多关系。
- `article.id` 与 `article_like.article_id` 为一对多关系。
- `article.id` 与 `article_favorite.article_id` 为一对多关系。
