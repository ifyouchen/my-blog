# my-blog 数据库设计

## 1. 设计说明

数据库使用 MySQL 8.0+，InnoDB 引擎，`utf8mb4` 字符集。所有业务表采用逻辑删除和乐观锁。

### 通用字段

所有业务表必须包含以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `bigint unsigned NOT NULL AUTO_INCREMENT` | 主键 |
| `created_at` | `datetime NOT NULL DEFAULT CURRENT_TIMESTAMP` | 创建时间 |
| `updated_at` | `datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP` | 更新时间 |
| `deleted_at` | `datetime DEFAULT NULL` | 删除时间（软删除） |
| `version` | `int NOT NULL DEFAULT 0` | 乐观锁版本号 |

软删除统一使用 `deleted_at IS NULL` 判断，禁止物理删除。

## 2. 状态枚举

### 2.1 用户状态

| 值 | 含义 |
|------|------|
| `NORMAL` | 正常 |
| `DISABLED` | 禁用 |
| `DELETED` | 已删除 |

### 2.2 用户角色

| 值 | 含义 |
|------|------|
| `USER` | 普通用户 |
| `ADMIN` | 管理员 |

### 2.3 文章状态

| 值 | 含义 |
|------|------|
| `DRAFT` | 草稿 |
| `SCHEDULED` | 定时发布 |
| `PUBLISHED` | 已发布 |
| `OFFLINE` | 已下架 |
| `DELETED` | 已删除 |

### 2.4 通知类型

`ARTICLE_LIKE`, `ARTICLE_FAVORITE`, `ARTICLE_COMMENT`, `COMMENT_REPLY`, `COMMENT_LIKE`, `USER_FOLLOW`, `ARTICLE_PUBLISH`, `REPORT_SUBMITTED`, `RECOMMENDATION_APPLIED`

## 5. 关系说明

- `blog_user.id` 与 `blog_article.author_id` 为一对多关系
- `blog_article` 与 `blog_tag` 通过 `blog_article_tag` 建立多对多关系（基于 tag_name）
- `blog_category_group.id` 与 `blog_category.group_id` 为一对多关系
- `blog_article.id` 与 `blog_comment.article_id` 为一对多关系
- `blog_article.id` 与 `blog_article_like/favorite` 为一对多关系
- `blog_user.id` 与 `blog_user_follow.follower/following` 为一对多关系
- `blog_column.id` 与 `blog_column_article.column_id` 为一对多关系
- `blog_topic.id` 与 `blog_topic_article.topic_id` 为一对多关系
- `blog_conversation.id` 与 `blog_message.conversation_id` 为一对多关系
- 成长模块表通过 `user_id` 关联 `blog_user.id`
