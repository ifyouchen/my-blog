# 博客网站第一版权限设计

## 1. 权限目标

权限设计用于确保不同用户只能访问和操作自己被允许的数据。第一版采用 JWT 完成身份认证，后端根据用户角色和资源归属进行授权判断。

## 2. 角色定义

| 角色 | 说明 |
|---|---|
| 游客 | 未登录用户。 |
| 普通用户 | 已登录用户，角色值为 `USER`。 |
| 作者 | 发布过文章的普通用户，不作为独立角色存储。 |
| 管理员 | 后台管理用户，角色值为 `ADMIN`。 |

## 3. 用户状态

| 状态 | 说明 |
|---|---|
| `NORMAL` | 可以正常登录和操作。 |
| `DISABLED` | 被禁用，不能登录和操作内容。 |
| `DELETED` | 已删除，不允许登录。 |

## 5. 接口权限规则

### 5.1 无需登录的接口

- `GET /api/articles`
- `GET /api/articles/{id}`
- `GET /api/articles/{articleId}/comments`
- `GET /api/categories`
- `GET /api/tags`
- `GET /api/users/{id}`
- `POST /api/auth/register`
- `POST /api/auth/login`

### 5.2 需要登录的接口

- `POST /api/auth/logout`
- `GET /api/users/me`
- `PUT /api/users/me`
- `POST /api/articles`
- `GET /api/users/me/articles`
- `POST /api/articles/{id}/like`
- `DELETE /api/articles/{id}/like`
- `POST /api/articles/{id}/favorite`
- `DELETE /api/articles/{id}/favorite`
- `GET /api/users/me/favorites`
- `POST /api/articles/{articleId}/comments`
- `POST /api/uploads/images`

### 5.3 需要资源归属校验的接口

- `PUT /api/articles/{id}`：文章作者或管理员。
- `DELETE /api/articles/{id}`：文章作者或管理员。
- `DELETE /api/comments/{id}`：评论作者、文章作者或管理员。

### 5.4 仅管理员接口

- `/api/admin/**`

## 6. 文章权限规则

- 草稿文章只允许作者本人和管理员查看。
- 已发布文章允许所有用户查看。
- 已下架文章只允许作者本人和管理员查看。
- 已删除文章默认不允许前台查看，管理员可在后台审计时查看基础信息。
- 文章编辑和删除必须校验当前用户是否为作者本人或管理员。

## 7. 评论权限规则

- 游客可以查看评论。
- 登录用户可以发表评论和回复评论。
- 评论作者可以删除自己的评论。
- 文章作者可以删除自己文章下的评论。
- 管理员可以删除任意评论。
- 被删除评论不在前台展示。

## 8. 点赞收藏权限规则

- 游客不能点赞和收藏。
- 登录用户不能重复点赞同一文章。
- 登录用户不能重复收藏同一文章。
- 已删除或已下架文章不能新增点赞和收藏。
- 取消点赞或取消收藏只能操作自己的记录。

## 9. 用户禁用规则

用户状态为 `DISABLED` 时：

- 不能登录。
- 不能发布文章。
- 不能编辑文章。
- 不能发表评论。
- 不能点赞。
- 不能收藏。
- 不能上传文件。

如果用户在登录后被禁用，后端在每次校验 JWT 后仍需查询用户当前状态，避免旧 Token 继续操作。

## 10. 管理员操作规则

- 管理员可以禁用和启用用户。
- 管理员可以下架和恢复文章。
- 管理员可以删除评论。
- 管理员可以维护分类和标签。
- 管理员的关键操作需要记录到 `admin_log`。
- 管理员不能通过后台查看用户明文密码。

## 11. 前端路由控制

- 未登录用户访问需要登录的页面时跳转登录页。
- 普通用户访问 `/admin` 路由时跳转无权限页面或首页。
- 前端路由控制只用于改善体验，最终权限必须由后端接口校验。
