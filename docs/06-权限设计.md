# 博客网站权限设计

## 1. 权限目标

权限设计确保不同用户只能访问和操作自己被允许的数据。项目采用自定义 JWT（HMAC-SHA256）完成身份认证，后端通过 `AuthContext` 获取当前用户信息，在 Controller 和 Application Service 中手动进行角色和归属校验。**不使用 Spring Security**。

## 2. 角色定义

| 角色 | 存储值 | 说明 |
|------|--------|------|
| 游客 | 无 Token | 未登录用户，仅可访问公开接口 |
| 普通用户 | `USER` | 已登录用户，可发文、评论、点赞、收藏、关注 |
| 管理员 | `ADMIN` | 后台管理用户，可访问 `/api/admin/**` 所有接口 |

项目不区分"作者"为独立角色——发表过文章的用户仍为 `USER` 角色，文章归属校验通过作者 ID 比对完成。

## 3. 用户状态

| 状态 | 说明 |
|------|------|
| `NORMAL` | 正常状态，可正常登录和操作 |
| `DISABLED` | 被禁用，不能登录和操作内容 |
| `DELETED` | 已删除（软删除），不允许登录 |

每次 JWT 认证时后端都会查用户状态（缓存 5 分钟），即使 Token 有效也会拒绝禁用/删除用户的请求。

## 4. 认证架构

```
Client                          Server (JwtAuthenticationInterceptor)
  │                                    │
  │  Request + Authorization: Bearer   │
  │ ──────────────────────────────────→│
  │                                    │ 解析 JWT (HS256)
  │                                    │ 验证签名、过期时间
  │                                    │ 查询用户状态缓存 (Redis 5min TTL)
  │                                    │ 设置 AuthContext (ThreadLocal)
  │                                    │ → Controller 处理
  │                                    │ → AuthContext.clear() 在 afterCompletion
  │ ← Response                        │
```

- 公开接口：Token 可选，如有有效 Token 则解析用户信息用于体验优化（如点赞状态）。
- 受保护接口：必须携带有效 Token。
- 管理员接口：除 Token 外还需校验 `AuthContext.getRole() == "ADMIN"`。

## 6. 接口权限分类

### 6.1 完全公开（无 Token 要求）

- `GET /api/auth/register/email-code`、`POST /api/auth/register`、`POST /api/auth/login`
- `POST /api/auth/password/forgot`、`POST /api/auth/password/reset`
- `GET /api/health`
- `GET /robots.txt`、`/sitemap.xml`、`/rss.xml`

### 6.2 公开读 + 可选认证

以下 GET 接口允许匿名访问，但若请求携带有效 Token 则额外填充当前用户的互动状态：

- 文章列表/详情/邻居/验证：`/api/articles/**`
- 分类/标签/专栏/专题列表：`/api/categories/**`、`/api/tags/**`、`/api/columns/**`、`/api/topics/**`
- 首页数据：`/api/home/**`
- 排行榜：`/api/rankings/**`
- 推荐：`/api/recommendations/**`
- 搜索：`/api/search/**`
- 用户主页：`/api/users/{id}/**`
- 公告：`/api/announcements/active`
- 广告展示：`/api/ads/**`
- 敏感词提示：`/api/sensitive-words/warn`

### 6.3 需登录（校验 Token + 用户状态）

- 文章创建/更新/删除/解锁规则/版本管理
- 评论创建/编辑/删除/点赞/置顶
- 点赞/收藏
- 关注/取关
- 用户资料修改/密码修改/邮箱换绑
- 文件上传（图片/文件/预签名 URL）
- 通知/消息 CRUD + SSE 流
- 创作后台（Dashboard）所有接口
- 积分/经验/签到/徽章/邀请码
- 文章解锁/收益查询

### 6.4 需资源归属校验

- `PUT /api/articles/{id}` / `DELETE /api/articles/{id}`：文章作者或管理员
- `PUT /api/comments/{id}`：评论作者（仅 10 分钟内可编辑）
- `DELETE /api/comments/{id}`：评论作者、文章作者或管理员
- `POST /api/comments/{id}/pin`：文章作者或管理员
- `PUT /api/columns/{id}` / `DELETE /api/columns/{id}`：专栏作者

### 6.5 仅管理员

- `/api/admin/**` 全部接口
- `/api/admin/points/**`
- `/api/admin/growth/**`
- `/api/admin/rewards/**`
- `/api/admin/revenue/**`
- `/api/admin/recommendation-applications/**`
- `/api/categories` 的写操作（`POST`/`PUT`/`DELETE`）
- `/api/tags` 的写操作（`POST`/`PUT`/`DELETE`）

### 6.6 开放接口（无认证，用于外部回调）

- `POST /api/open/invite/reward/trigger`：邀请奖励内部触发
- `POST /api/open/pay/recharge/callback`：支付回调

## 7. 文章权限规则

- **草稿 (DRAFT)**：仅作者和管理员可查看。
- **已发布 (PUBLISHED)**：所有用户可见。
- **已下线 (OFFLINE)**：仅作者和管理员可查看。
- **已删除 (DELETED)**：前台不可见，管理员可在后台审计时查看基础信息。
- **付费文章**：未解锁用户仅可见标题和摘要，正文被遮罩覆盖，需消耗积分解锁后查看全文。
- 文章编辑/删除必须校验当前用户为作者或管理员。

## 8. 评论权限规则

- 游客可查看评论，不可发表。
- 登录用户可发表评论和回复评论。
- 评论作者可编辑自己的评论（10 分钟内，编辑次数限制）。
- 评论作者可删除自己的评论。
- 文章作者和管理员可删除任意评论（置顶评论也可删除）。
- 文章作者和管理员可置顶/取消置顶评论。
- 被删除评论在前台隐藏。

## 9. 点赞收藏权限规则

- 游客不能点赞和收藏。
- 同一用户不能重复点赞同一文章/评论（幂等）。
- 同一用户不能重复收藏同一文章（幂等）。
- 已删除或已下线文章不能新增点赞和收藏。
- 取消点赞/收藏只能操作自己的记录。

## 10. 用户禁用规则

用户状态为 `DISABLED` 时：

- 不能登录（登录接口返回登录失败）。
- Token 持有者已登录后被禁用，下次请求被拦截（5 分钟缓存生效）。
- 不能发布/编辑文章，不能发表评论，不能点赞/收藏，不能上传文件。

## 11. 管理员操作规则

- 管理员可以禁用/启用用户、修改用户角色。
- 管理员可以下架/恢复/精选/删除文章（支持批量）。
- 管理员可以删除评论、审核评论。
- 管理员可以维护分类/标签/专栏/专题/广告/公告/敏感词。
- 管理员可以处理举报（拒绝/下架文章/删除评论/禁用用户）。
- 管理员可以配置经验/积分规则、管理员调分、查询分账。
- 管理员的关键操作记录到 `admin_log`（含操作前后快照）。
- 管理员不能通过后台查看用户明文密码。

## 12. 前端路由控制

- 未登录用户访问 `meta.requiresAuth` 页面时弹出登录弹窗，不跳转页面。
- 非管理员用户访问 `meta.requiresAdmin` 页面时重定向到 `/403`。
- 前端路由控制仅改善体验，最终权限由后端接口校验。

## 13. 幂等与安全约束

- 所有敏感请求（注册/登录/忘记密码/上传）均有 Redis 限流（默认每 IP 每分钟 20 次）。
- 图片上传单独限流：每 IP 每 60 秒 10 次。
- 文件上传单独限流：每 IP 每 60 秒 5 次。
- 邮件发送限流：IP + 邮箱双重频率限制。
- JWT 密钥生产环境必须使用长随机字符串。
- 密码使用 BCrypt 哈希，禁止明文存储。
