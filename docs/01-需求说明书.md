# my-blog 博客平台需求说明书

> 版本：v1.0.0  
> 定位：面向技术创作者和技术读者的博客社区平台

---

## 1. 项目背景

`my-blog` 是一个面向技术创作者和技术读者的博客社区，采用前后端分离架构。核心能力覆盖用户注册登录、文章发布浏览、评论互动、点赞收藏关注、专栏专题、私信通知、创作者数据面板、管理后台以及用户成长体系（经验/积分/等级/徽章/签到/邀请/付费解锁）。

后端采用 DDD 四层架构（Interfaces → Application → Domain → Infrastructure），前端采用 Vue 3 + Vite。

## 2. 项目目标

- 为用户提供稳定的技术文章发布和阅读体验
- 为创作者提供文章管理、数据反馈、专栏组织和成长激励
- 为平台管理员提供全面的运营管理能力
- 建立基础的用户激励与商业变现体系

## 3. 用户角色

| 角色 | 说明 |
|------|------|
| 游客 | 未登录用户，可浏览文章、搜索、查看评论 |
| 普通用户 | 已登录用户，可评论、点赞、收藏、关注、发私信，无单独作者角色（发文的用户即为作者） |
| 创作者 | 已发布文章的用户，可通过创作者面板管理文章、专栏、查看数据 |
| 管理员 | 平台管理人员，可管理用户、文章、评论、分类、标签、专栏、专题、广告、公告、敏感词、举报、操作日志和成长配置 |

## 4. 功能范围

### 4.1 用户系统

- 邮箱注册（需验证码）
- 邮箱/用户名 + 密码登录
- 退出登录
- 忘记密码（邮箱验证码重置）
- 个人资料修改（头像、昵称、简介、网站、GitHub、Twitter、所在地）
- 修改密码
- 换绑邮箱（新旧双重验证）
- 账户安全信息（登录记录）
- 公开个人主页

### 4.2 文章系统

- 富文本编辑（TipTap 编辑器，支持标题、加粗、斜体、引用、列表、代码块、表格、任务列表、图片、链接）
- 代码语法高亮（highlight.js + lowlight，支持多语言）
- Markdown 渲染预览（markdown-it）+ HTML 转 Markdown（Turndown）
- 文章封面上传
- 保存草稿（DRAFT）
- 发布文章（DRAFT → PUBLISHED，记录发布时间）
- 定时发布（SCHEDULED → PUBLISHED）
- 下线文章（PUBLISHED → OFFLINE）
- 删除文章（软删除）
- 文章版本管理（每次编辑记录版本，支持历史版本查看和回滚）
- 文章导出（ZIP 含图片）
- 文章目录/大纲自动生成
- 付费文章积分解锁（need_unlock / unlock_point_price）
- 阅读量、点赞、收藏、评论计数
- 邻居文章（上下篇）、推荐文章、相关文章

### 4.3 分类与标签

- 分组分类体系（category_group → category）
- 标签打标（支持热门标签）
- 按分类、标签浏览文章
- 管理端分类/标签 CRUD

### 4.4 专栏系统

- 创作者创建和管理专栏
- 专栏封面、简介、难度标签
- 专栏文章排序（支持章节和学习路径）
- 专栏订阅

### 4.5 专题系统

- 平台创建和管理专题
- 专题封面、简介、难度、学习路径
- 专题内文章排序

### 4.6 互动系统

- 文章评论与回复（支持编辑、点赞、置顶）
- 文章点赞/取消点赞
- 文章收藏/取消收藏
- 用户关注/取消关注
- 关注动态 Feed

### 4.7 消息与通知

- 系统通知（评论、点赞、收藏、关注等事件，支持 SSE 实时推送）
- 私信（基于会话的聊天，支持 SSE 实时推送未读计数）
- 通知已读/全部已读
- 公告展示

### 4.8 内容发现

- 首页 Bootstrap 综合推荐
- 统一搜索（文章、专栏、话题硬搜索）
- 热门/近期搜索关键词
- 文章排行榜（热门、最新）
- 作者排行榜
- 个性化推荐（推荐文章、专栏、作者）
- 浏览历史记录

### 4.9 创作者后台（Dashboard）

- 数据概览（文章数、阅读量、点赞数、评论数、粉丝数统计 + 趋势图）
- 我的文章管理（状态筛选、编辑、删除）
- 我的收藏浏览
- 我的专栏管理（创建、编辑、删除、管理文章）
- 成长面板（经验、等级、积分、徽章、签到）

### 4.10 管理后台（Admin）

- 全站统计面板（用户数、文章数、评论数、趋势）
- 用户管理（搜索、状态管理、角色管理、推荐设置）
- 文章管理（全站文章审核、状态变更、精选、批量操作）
- 评论管理（审核、删除）
- 举报处理（审核、下线、禁用等处理动作）
- 分类管理（分类组 + 分类 CRUD）
- 标签管理（标签 CRUD）
- 专栏管理（全站专栏 CRUD + 文章管理）
- 专题管理（专题 CRUD + 文章管理）
- 广告管理（广告位 + 投放 CRUD，曝光/点击统计）
- 公告管理（公告 CRUD + 发布/取消发布）
- 敏感词管理（WARN/BLOCK 两级）
- 管理员操作日志（审计）
- 内容推荐管理（精选文章、推荐用户）
- 首页推荐申请审核
- 数据导出（文章、用户）

### 4.11 用户成长体系（Growth 模块）

- **经验值（EXP）**：按行为事件（点赞、阅读、评论、收藏、分享、关注、发布）获取经验
- **等级体系**：8 级等级制（新手 → 传奇），每级对应不同经验阈值
- **积分系统**：可消费的虚拟货币
- **每日签到**：连续签到阶梯奖励（最高 60 积分/天）
- **徽章系统**：成就徽章，可装备展示
- **邀请奖励**：邀请注册奖励 50 积分
- **充值入账**：1 元 = 1 积分
- **文章解锁**：消耗积分解锁付费文章
- **收益分成**：平台:作者 = 50:50 分账
- **创作者特权**：等级解锁不同特权

### 4.12 文件/图片上传

- 云存储（COS）预签名 URL 直传，前端不接触密钥
- 开发环境降级为后端代理上传
- 上传限流（每 IP 每 60 秒 10 次）
- 文件类型和大小校验（20MB/文件，22MB/请求）

## 5. 非目标范围

以下能力不纳入当前版本：

- 第三方登录（OAuth）
- Elasticsearch 全文搜索（当前使用 MySQL FULLTEXT）
- 移动 App / 小程序
- 多租户或企业空间
- 复杂运营活动

## 6. 核心业务流程

### 6.1 注册登录流程

```
用户提交注册 → 邮箱验证码校验 → 后端创建用户 → 返回用户信息
用户提交登录 → 后端校验密码(BCrypt) → 生成 JWT → 返回 Token + 用户信息
前端保存 Token → 后续请求放入 Authorization 头 → 拦截器校验 JWT + 用户状态
```

### 6.2 发布文章流程

```
用户登录 → 进入编辑器 → 填写标题/正文/分类/标签/封面 → 保存草稿或发布
发布 → DRAFT → PUBLISHED → 记录 published_at → 触发成长事件
```

### 6.3 文章生命周期

```
DRAFT ──发布──→ PUBLISHED ──下线──→ OFFLINE
  ↑                │                    │
  └──── 重新发布 ──┘                    │
                                        ↓
                                    DELETED(软删除)
```

### 6.4 浏览与互动流程

```
文章详情页加载 → 若需解锁且未登录 → 显示遮罩
若已登录 → 查询解锁状态 → 已解锁展示正文 / 未解锁显示解锁引导
用户可评论/点赞/收藏/关注作者
```

### 6.5 经验积分增长流程

```
用户行为(发文/评论/点赞/签到等) → 事件触发
  → GROWTH_RULE_CONFIG 匹配
  → 经验值变更(user_growth_account)
    → 等级升级检测 → 等级特权解锁
  → 积分变更(user_point_account)
    → 可消费：文章解锁、推荐申请等
```

### 6.6 后台审核流程

```
管理员登录后台 → 查看用户/文章/评论/举报列表
→ 执行下架/删除/禁用/审核通过 → 系统记录 admin_log
```

## 7. 数据范围

- 文章正文使用 Markdown 格式存储
- 所有业务表使用软删除（deleted_at IS NULL）
- 所有表使用乐观锁（version）
- 密码使用 BCrypt 加密
- 认证使用 JWT（24h 过期）
- 所有表使用 utf8mb4 字符集、InnoDB 引擎

## 8. 验收标准

- 用户可以完成注册、登录、退出、密码重置和个人资料修改
- 用户可发布、编辑、删除自己的文章（含草稿和定时发布）
- 游客可浏览首页、文章列表、文章详情和搜索结果
- 登录用户可评论、点赞、收藏、关注
- 用户不能编辑或删除别人的文章
- 管理员可通过后台管理所有资源
- 被禁用用户不能登录和操作内容
- 经验/积分/签到/解锁等成长功能正常运行
- SSE 实时推送通知和消息
- 所有受保护接口校验 JWT 和用户状态
- 所有操作日志可审计追溯
