## 1. 身份认证链路

### 1.1 注册链路

**入口**：`POST /api/auth/register`

```
客户端 POST {username, email, password, emailCode, inviteCode?}
       │
       ▼
 JwtAuthenticationInterceptor
 ──► 白名单路径，直接放行（无需 Token）
       │
       ▼
 AuthController.register()
 ──► RestDtoMapper.toCommand(RegisterRequest) → RegisterCommand
       │
       ▼
 AuthAppService.register(RegisterCommand)
       │
       ├─[1] 邮箱验证码校验
       │      RegisterEmailCodeAppService.verify(email, code)
       │      ├─ 查缓存中该邮箱的验证码记录
       │      ├─ 验证码不存在 → 抛 PARAM_ERROR "验证码不存在或已过期"
       │      ├─ 验证码不匹配 → 抛 PARAM_ERROR "验证码错误"
       │      └─ 验证码已过期（5分钟）→ 抛 PARAM_ERROR "验证码已过期"
       │
       ├─[2] 邀请码校验（可选，如系统开启）
       │      InviteCodeAppService.validate(inviteCode)
       │      └─ 无效 → 抛 PARAM_ERROR
       │
       ├─[3] 唯一性校验
       │      UserRepository.existsByUsername(username)  → 存在 → 抛 "用户名已存在"
       │      UserRepository.existsByEmail(email)        → 存在 → 抛 "邮箱已被注册"
       │
       ├─[4] 密码加密
       │      PasswordDomainService.encode(password) → BCrypt Hash
       │
       ├─[5] 创建用户聚合根
       │      User.create(id, username, email, passwordHash)
       │      └─ 设置默认值：nickname=username, avatarUrl=Unsplash默认图, bio="热爱技术写作"
       │         role=USER, status=NORMAL, createdAt=now()
       │
       ├─[6] 持久化
       │      UserRepository.save(user)
       │
       ├─[7] 签发 JWT
       │      JwtTokenProvider.createToken(userId, username, role)
       │
       └─ 返回 AuthResponse { token, userInfo }
```

**异常路径**：
- 用户名 < 3 字符 → `DomainException(PARAM_ERROR, "用户名至少3个字符")`
- 密码为空 → `DomainException(PARAM_ERROR, "密码不能为空")`

### 1.2 登录链路

**入口**：`POST /api/auth/login`

```
客户端 POST {account(用户名或邮箱), password}
       │
       ▼
 AuthAppService.login(LoginCommand)
       │
       ├─[1] 查找用户（两路并行判断）
       │      account 含 '@' ──► UserRepository.findByEmail(account)
       │      account 不含 '@' ──► UserRepository.findByUsername(account)
       │      ├─ 未找到 → 抛 "账号或密码错误"（不区分是用户名还是密码错误，防枚举）
       │      └─ 找到 → User 对象
       │
       ├─[2] 密码校验
       │      BCrypt.matches(rawPassword, user.passwordHash)
       │      └─ 不匹配 → 抛 "账号或密码错误"
       │
       ├─[3] 账号状态校验
       │      user.ensureCanLogin()
       │      └─ status != NORMAL → 抛 FORBIDDEN "账号不可用"
       │
       ├─[4] 记录登录信息
       │      user.recordLogin(ip)
       │      └─ 更新 lastLoginAt = now(), lastLoginIp = ip
       │
       ├─[5] 持久化
       │      UserRepository.save(user)
       │
       ├─[6] 签发 JWT
       │      JwtTokenProvider.createToken(userId, username, role)
       │
       └─ 返回 AuthResponse { token, userInfo }
```

### 1.3 密码找回与重置链路

**入口**：`POST /api/auth/password/forgot` + `POST /api/auth/password/reset`

```
── 步骤一：申请重置 ──────────────────────────────────
客户端 POST /api/auth/password/forgot {email}
       │
       ▼
 AuthAppService.forgotPassword(email)
       │
       ├─[1] 查找用户 → 不存在 → 抛 "该邮箱未注册"
       │
       ├─[2] 生成重置 Token
       │      user.generatePasswordResetToken()
       │      ├─ 生成 32 位 UUID（无横线）
       │      └─ 设置过期时间 expireAt = now() + 30分钟
       │
       ├─[3] 持久化 token
       │      UserRepository.save(user)
       │
       └─[4] 发送重置邮件（异步）
              EmailQueueAppService.enqueue(PASSWORD_RESET, email, {resetLink})
              resetLink = {frontendBaseUrl}/reset-password?token={token}

── 步骤二：重置密码 ──────────────────────────────────
客户端 POST /api/auth/password/reset {token, newPassword}
       │
       ▼
 AuthAppService.resetPassword(token, newPassword)
       │
       ├─[1] 通过 token 查找用户
       │      UserRepository.findByPasswordResetToken(token) → 不存在 → 抛 "无效链接"
       │
       ├─[2] 校验 token 有效性
       │      user.validatePasswordResetToken(token)
       │      ├─ token 不匹配 → 抛 "重置链接无效或已过期"
       │      └─ expireAt 已过 → 抛 "重置链接已过期，请重新申请"
       │
       ├─[3] 加密新密码 + 更新
       │      newHash = BCrypt.encode(newPassword)
       │      user.changePassword(newHash)
       │      └─ 同时清空 passwordResetToken + passwordResetExpire
       │
       └─[4] 持久化
              UserRepository.save(user)
```

### 1.4 请求认证拦截链路

每个 API 请求（除白名单外）都经过以下处理：

```
HTTP 请求
    │
    ▼
RequestTimingFilter（OncePerRequestFilter，优先级最高）
    ├─ 读取请求头 X-Trace-Id
    ├─ 存在且非空 → 透传，设置 MDC.put("traceId", existingId)
    └─ 不存在 → 生成 UUID.randomUUID().toString().replace("-","").substring(0,12)
               设置到 MDC 和响应头 X-Trace-Id
    │
    ▼（finally 块保证清理）
    └─ filterChain.doFilter() 后 MDC.remove("traceId")

    │
    ▼
JwtAuthenticationInterceptor.preHandle()
    │
    ├─ METHOD = OPTIONS → 直接 return true（CORS 预检放行）
    │
    ├─ 读取 header = Authorization，queryToken = ?token=
    │
    ├─ isPublicRequest(request)?
    │       │
    │    YES │
    │       ├─ authenticateIfPresent(header, queryToken, lenient=true)
    │       │    ├─ header 含 "Bearer " → 截取 token → authenticateToken(token, lenient=true)
    │       │    │       ├─ 解析成功 → AuthContext.set(payload) → return true
    │       │    │       └─ ApplicationException → 静默 return false（宽松模式）
    │       │    ├─ queryToken 非空 → 同上逻辑
    │       │    └─ 无 token → return false
    │       └─ return true（无论认证结果）
    │
    └─ isPublicRequest = false（受保护接口）
            ├─ authenticateIfPresent(header, queryToken, lenient=false)
            │       ├─ 认证成功 → AuthContext.set(payload) → return true
            │       └─ 认证失败 → 直接抛出 ApplicationException（严格模式）
            └─ 无 token → 抛 UNAUTHORIZED "请先登录"

    │
    ▼
authenticateToken() 内部：
    ├─ JwtTokenProvider.parseToken(token)
    │       ├─ 分割为3段，不是3段 → 抛 "Token格式错误"
    │       ├─ Base64解码 header，检查含 "HS256" → 否则抛 "不支持的Token算法"
    │       ├─ 重新签名比对 → 不匹配 → 抛 "Token签名无效"
    │       ├─ 解码 payload → 反序列化为 JwtPayload
    │       └─ payload.expireAt < now() → 抛 "Token已过期"
    │
    └─ ensureUserAvailable(userId)
            ├─ UserRepository.findById(userId) → 不存在 → 抛 "请先登录"
            └─ user.status != NORMAL → 抛 FORBIDDEN "账号不可用"

    │
    ▼（请求处理完成）
afterCompletion()
    └─ AuthContext.clear()（ThreadLocal.remove()，防止线程复用污染）
```

## 2. 文章链路

### 2.1 创建文章链路

**入口**：`POST /api/articles`

```
ArticleAppService.create(CreateArticleCommand, userId)
    │
    ├─[1] 敏感词检测
    │      SensitiveWordAppService.check(title + content)
    │      ├─ 发现高级别敏感词 → 拒绝保存
    │      └─ 发现低级别 → 设置 warnFlag=true，仍允许保存（标记待审核）
    │
    ├─[2] 获取下一个可用 ID
    │      ArticleRepository.nextId()
    │
    ├─[3] 分类校验（若指定分类）
    │      CategoryAppService.getByName(category)
    │      └─ 不存在 → 抛 "分类不存在"
    │
    ├─[4] 标签同步
    │      TagAppService.syncTags(tagNames)
    │      └─ 新标签自动创建，已有标签复用，useCount 累加
    │
    ├─[5] 按 status 分发创建逻辑
    │
    │   ── DRAFT ─────────────────────────────────────────
    │      Article.saveDraft(id, authorId, title, summary, content, ...)
    │      └─ 无必填校验，title/content 可为空
    │
    │   ── PUBLISHED ──────────────────────────────────────
    │      Article.publish(...)
    │      ├─ title 为空 → 抛 "请输入文章标题"
    │      ├─ content 为空 → 抛 "请输入文章内容"
    │      ├─ category 为空 → 抛 "请选择文章分类"
    │      └─ 设置 publishedAt = now()
    │
    │   ── SCHEDULED ──────────────────────────────────────
    │      Article.schedulePublish(id, ..., scheduledPublishAt)
    │      ├─ scheduledPublishAt 为空 → 抛 "请选择定时发布时间"
    │      └─ scheduledPublishAt <= now() → 抛 "定时发布时间必须晚于当前时间"
    │
    ├─[6] 持久化
    │      ArticleRepository.save(article)
    │      └─ 同步保存 article_tag 关联记录
    │
    ├─[7] 异步保存版本快照
    │      ArticleVersionRepository.save(versionSnapshot)
    │
    └─[8] 缓存失效（异步）
           HomePortalCacheInvalidator.invalidate()
           ├─ homeStatsCache.invalidateAll()
           ├─ homeBootstrapCache.invalidateAll()
           └─ featuredArticlesCache.invalidateAll()
```

### 2.2 编辑与更新文章

**入口**：`PUT /api/articles/{id}`

```
ArticleAppService.update(id, command, userId)
    │
    ├─[1] 查找文章
    │      ArticleRepository.findById(id) → 不存在 → 抛 "文章不存在"
    │
    ├─[2] 权限校验

    ├─[3] 状态校验
    │      status = DELETED → 抛 "文章已删除"
    │
    ├─[4] 敏感词检测（同创建）
    │
    ├─[5] 调用聚合根更新方法
    │      article.updateContent(title, summary, content, coverUrl, category, tags, slug, seo...)
    │      ├─ updatedAt = now()
    │      └─ version += 1（乐观锁递增）
    │
    ├─[6] 持久化
    │
    └─[7] 异步保存版本快照 + 缓存失效
```

### 2.3 发布/下架/删除文章

```
── 发布（草稿→已发布）──────────────────────────────────

    │
    ArticleAppService.publish(id, userId)
    ├─ 查文章，校验权限

    │

    ├─ article.offline(reason)
    │   └─ status = OFFLINE, offlineReason = reason

DELETE /api/articles/{id}
    │
    ArticleAppService.delete(id, userId)
    ├─ 校验权限
    ├─ article.delete()
    │   └─ status = DELETED, deletedAt = now()
    └─ 持久化
```

## 3. 评论链路

### 3.1 发表评论链路

**入口**：`POST /api/articles/{articleId}/comments`

```
CommentAppService.createComment(CreateCommentCommand, role)
    │
    ├─[1] 查找文章
    │      ArticleRepository.findById(articleId)
    │      ├─ 不存在 → 抛 "文章不存在"
    │      └─ status != PUBLISHED → 抛 "文章不存在或已下架"
    │
    ├─[2] 若为回复（parentId 非空）
    │      CommentRepository.findById(parentId)
    │      ├─ 不存在 → 抛 "父评论不存在"
    │      ├─ parentComment.articleId != articleId → 抛 "评论不属于该文章"
    │      └─ 确定 rootCommentId（若父评论是一级则 root=parentId，否则 root=parent.rootCommentId）
    │
    ├─[3] 敏感词检测
    │      SensitiveWordAppService.check(content)
    │
    ├─[4] 创建评论聚合根
    │      Comment.create(id, articleId, userId, rootCommentId, parentId, content)
    │      └─ content 长度 > 1000 → 抛 "评论内容不能超过1000字"
    │
    ├─[5] 持久化
    │      CommentRepository.save(comment)
    │
    ├─[6] 更新文章评论数
    │      article.increaseCommentCount()
    │      ArticleRepository.save(article)
    │
    └─[7] 异步发送通知
           ├─ 一级评论（parentId=null） → 通知文章作者(ARTICLE_COMMENT)
           └─ 回复评论 → 通知被回复者(COMMENT_REPLY)
              （注：若回复者是文章作者本人则合并为一条通知）
```

### 3.2 评论编辑与删除

```
── 编辑（10分钟窗口）──────────────────────────────────
PUT /api/comments/{id}
    │
    CommentAppService.editComment(id, newContent, userId)
    │
    ├─ 查找评论，校验是否是本人
    ├─ comment.edit(newContent)
    │   ├─ now() - comment.createdAt > 10分钟 → 抛 "超过编辑时间窗口"
    │   ├─ status != PUBLISHED → 抛 "无法编辑该评论"
    │   ├─ newContent 为空 → 抛 "评论内容不能为空"
    │   └─ 更新 content, editedAt=now(), editCount+=1
    └─ 持久化

── 删除 ──────────────────────────────────────────────
DELETE /api/comments/{id}
    │
    CommentAppService.deleteComment(id, userId, role)
    │
    ├─ 查找评论
    ├─ 权限校验: comment.userId == userId OR role == ADMIN
    ├─ comment.delete()
    │   └─ status = DELETED, deletedAt = now()
    ├─ 持久化
    └─ 异步更新文章 commentCount - 1
```

---

### 3.3 评论置顶

```
POST /api/comments/{id}/pin
    │
    ├─ 查找评论
    ├─ 权限校验：comment 所属文章的 authorId == userId OR role == ADMIN
    ├─ 仅支持一级评论（rootCommentId == null），否则 → 抛 "只能置顶一级评论"
    ├─ 查询该文章是否已有其他置顶评论 → 有则先取消旧置顶
    ├─ comment.pin()
    │   └─ pinned=true, pinnedAt=now()
    └─ 持久化
```

### 3.4 评论查询（树形分页）

```
GET /api/articles/{articleId}/comments?page=1&pageSize=20&sort=latest

CommentAppService.pageArticleComments(articleId, page, pageSize, sort, userId, role)
    │
    ├─[1] 查询一级评论（rootCommentId IS NULL）
    │      排序策略:
    │      ├─ 置顶评论（pinned=true）排最前（按 pinnedAt DESC）
    │      └─ 其余按 sort:
    │           latest → createdAt DESC
    │           hot    → likeCount DESC, createdAt DESC
    │
    ├─[2] 每条一级评论附带 replyPreview（最新3条回复）
    │      CommentRepository.findRecentReplies(rootCommentId, limit=3)
    │
    ├─[3] 若用户已登录
    │      CommentLikeRepository.existsActive(commentId, userId) → 填充 liked 字段
    │
    └─[4] 填充权限字段
           canDelete = (comment.userId == userId) || (role == ADMIN)
           canPin    = (article.authorId == userId) || (role == ADMIN)
           canEdit   = (comment.userId == userId) && (now() - createdAt <= 10min)
```

## 4. 互动链路

### 4.1 文章点赞链路

